  <name>Task 1: Run binary analysis and build independence verification</name>
  <files>asm/main.s</files>
  <action>
Perform a comprehensive automated verification of the build pipeline and binary correctness. Run the following checks IN ORDER and record the output:

**Step 1 -- Clean rebuild:**
```bash
make asm-clean
make asm
```
Verify: Exit code 0, no warnings or errors.

**Step 2 -- Binary type verification:**
```bash
file asm/bin/yetris-asm
```
Verify: Output contains "Mach-O 64-bit executable arm64".

**Step 3 -- Dynamic library dependencies:**
```bash
otool -L asm/bin/yetris-asm
```
Verify: Shows `/usr/lib/libncurses.*.dylib` and `/usr/lib/libSystem.B.dylib` (or their dyld cache equivalents). Should show exactly two library dependencies plus the binary itself.

**Step 4 -- Symbol resolution:**
```bash
nm -u asm/bin/yetris-asm
```
Verify: Lists undefined (external) symbols including `_initscr`, `_endwin`, `_cbreak`, `_noecho`, `_printw`, `_wrefresh`, `_wgetch`. All should be from ncurses or libSystem. No unexpected undefined symbols.

**Step 5 -- Defined symbols:**
```bash
nm asm/bin/yetris-asm | grep " T "
```
Verify: Shows `_main` in the TEXT section.

**Step 6 -- Build independence (C++ build still works):**
```bash
make clean
make
```
Verify: C++ build completes without errors, produces `bin/yetris`.

**Step 7 -- Cross-contamination check:**
```bash
make asm-clean
make asm
ls asm/bin/yetris-asm
ls bin/yetris
```
Verify: Both binaries exist. Assembly clean does not affect C++ binary, C++ clean does not affect assembly binary.

**Step 8 -- Source code ABI audit:**
Inspect asm/main.s to confirm:
- No use of x18 register anywhere
- Stack operations in multiples of 16
- Frame pointer setup present (x29)
- All external symbols prefixed with underscore
- GOT-indirect access for stdscr (@GOTPAGE/@GOTPAGEOFF)
- adrp+add for local string data (@PAGE/@PAGEOFF)
- .subsections_via_symbols present

If any check fails, diagnose the issue and fix asm/main.s. Common failure modes from research:
- SIGBUS at `bl` = stack misalignment → check stp/ldp pairs
- "Undefined symbols" linker error = missing underscore prefix → add `_` prefix
- Relocation errors about GOT = wrong addressing mode for external symbol → use @GOTPAGE/@GOTPAGEOFF
