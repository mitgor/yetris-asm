  <action>
Create directory `asm/` and file `asm/main.s` containing a complete AArch64 assembly ncurses hello-world program. Use the verified working pattern from the research document (01-RESEARCH.md, Code Examples section) as the foundation.

The source MUST demonstrate all 9 Darwin ABI conventions documented in the research:

1. **Underscore-prefixed symbols:** `_main` entry point, `_initscr`, `_endwin`, `_cbreak`, `_noecho`, `_printw`, `_wrefresh`, `_wgetch` -- all C-visible symbols prefixed with `_`.
2. **x16 for syscalls:** Not used directly (we use C library), but include a comment block documenting the convention for reference.
3. **svc #0x80:** Same as above -- document in comments, not used since we call C library functions.
4. **x18 reserved:** NEVER use x18 in any instruction. Use x9-x15 for temporaries, x19-x28 for callee-saved.
5. **16-byte stack alignment:** All stack allocations in multiples of 16. Use `stp x20, x19, [sp, #-32]!` + `stp x29, x30, [sp, #16]` pattern.
6. **Valid frame pointer (x29):** `add x29, sp, #16` after saving fp/lr pair. x29 always points to saved x29+x30 pair.
7. **adrp+add for local data:** Use `adrp x0, label@PAGE` + `add x0, x0, label@PAGEOFF` for string literals in __TEXT,__cstring.
8. **GOT-indirect for external globals:** Use `adrp xN, _stdscr@GOTPAGE` + `ldr xN, [xN, _stdscr@GOTPAGEOFF]` + `ldr x0, [xN]` to access stdscr from libncurses.
9. **Variadic args on stack:** Not needed for this hello-world (printw with no format args is a single-argument call), but include a comment noting the convention for future use.

Program flow:
1. Save callee-saved registers (x19, x20) + frame pointer + link register
2. Call `_initscr` to initialize ncurses
3. Call `_cbreak` to disable line buffering
4. Call `_noecho` to suppress input echo
5. Load address of hello string via adrp+add, call `_printw`
6. Load `_stdscr` via GOT indirection, call `_wrefresh` with WINDOW* in x0
7. Reload `_stdscr`, call `_wgetch` to wait for keypress
8. Call `_endwin` to restore terminal
9. Return 0 in w0, restore registers, `ret`

Use these Mach-O section directives:
- `.section __TEXT,__text,regular,pure_instructions` for code
- `.section __TEXT,__cstring,cstring_literals` for string data
- `.subsections_via_symbols` at end

Include `.globl _main` and `.p2align 2` before `_main`.

The hello string should be: `"yetris-asm: Hello from ARM64 assembly! Press any key to exit..."` (identifies the project in output).

Add a file header comment block with: project name, file purpose, build command, Darwin ABI convention summary (brief, 1-2 lines each for the 9 rules).

**Makefile changes:**

Append assembly build rules to the END of the existing Makefile (after the Commander stuff section, before the end of file). Use the verified Makefile pattern from the research document (01-RESEARCH.md, Pattern 3: Makefile Integration).

Add these variables and rules:

```makefile
# --- Assembly build (ARM64 on macOS Apple Silicon) ---
ASM_DIR     = asm
ASM_BIN_DIR = $(ASM_DIR)/bin
ASM_EXE     = $(ASM_BIN_DIR)/yetris-asm
ASM_SOURCES = $(wildcard $(ASM_DIR)/*.s)
ASM_OBJECTS = $(ASM_SOURCES:.s=.o)
SDK_PATH    = $(shell xcrun --show-sdk-path)

$(ASM_DIR)/%.o: $(ASM_DIR)/%.s
	# Assembling $<...
	$(MUTE)as -o $@ $<

$(ASM_EXE): $(ASM_OBJECTS) | $(ASM_BIN_DIR)
	# Linking assembly binary...
	$(MUTE)ld -o $@ $(ASM_OBJECTS) -lncurses -lSystem \
		-syslibroot $(SDK_PATH) -arch arm64

$(ASM_BIN_DIR):
	$(MUTE)mkdir -p $(ASM_BIN_DIR)

asm: $(ASM_EXE)
	# Assembly build successful!

asm-clean:
	# Cleaning assembly build...
	$(MUTE)rm -f $(ASM_OBJECTS) $(ASM_EXE)

asm-run: asm
	$(MUTE)./$(ASM_EXE)

.PHONY: asm asm-clean asm-run
```

Key design points (from research):
- `asm` target is completely independent of the `all` target -- they do not interfere
- Pattern rule `$(ASM_DIR)/%.o: $(ASM_DIR)/%.s` handles any number of `.s` files as the project grows
- `SDK_PATH` uses `xcrun --show-sdk-path` for portability across Xcode versions
- No `-e` flag needed because `_main` is the default entry point when linking with `-lSystem`
- Linker auto-signs the binary (no `codesign` step)
- Uses existing `$(MUTE)` variable for verbose mode consistency
- Order-only prerequisite `| $(ASM_BIN_DIR)` creates the output directory

**Do NOT modify any existing Makefile rules.** Only append new content.
