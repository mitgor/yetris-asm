**Step 2: Create asm/menu.s**

Create a new file `asm/menu.s` with the following exported functions. The Makefile's `ASM_SOURCES = $(wildcard $(ASM_DIR)/*.s)` auto-includes it.

The file structure follows the established patterns: file header comment, .section __TEXT,__text, .globl + .p2align 2 for each function, L-prefix local labels, stdscr via GOT in x19, callee-saved registers for state across bl calls, 16-byte aligned stacks.

**_menu_frame():** One frame of the menu state. Called from the state loop in main.s each iteration.
1. Load stdscr via GOT into x19 (callee-saved)
2. Call _werase(stdscr) to clear screen
3. Render title "Y E T R I S" at row 2, centered (col ~15 for 80-col terminal, or col 10 for simpler positioning). Use _waddstr.
4. Render 3 menu items (Start Game, Help, Quit) starting at row 5, col 10, spaced 2 rows apart (rows 5, 7, 9).
   - For each item: wmove to position, if index == _menu_selection then wattr_on(A_REVERSE=0x40000 constructed via `movz w1, #0x4, lsl #16`), waddstr the item string, then wattr_off if highlighted.
   - Use _menu_items pointer table to load string addresses: `adrp+add` to get _menu_items base, then `ldr x_reg, [base, index_lsl3]` for each entry.
5. Render 5 settings below the menu items starting at row 13, col 10:
   - Row 13: "Starting Level: NN" (render label via waddstr, then render number via _draw_number or manual digit output using the existing divide-by-10 pattern from render.s)
   - Row 15: "Ghost Piece:  ON/OFF"
   - Row 17: "Hold Piece:   ON/OFF"
   - Row 19: "Invisible:    OFF/ON"
   - Row 21: "Noise Rows:   NN"
   - Highlight the selected setting line with A_REVERSE if _menu_selection points to it (menu items 0-2 are actions, items 3-7 are settings, so _menu_selection range is 0-7 with items 0=Start, 1=Help, 2=Quit, 3=Level, 4=Ghost, 5=Hold, 6=Invisible, 7=Noise)
6. Call _wrefresh(stdscr)
7. Call _poll_input (returns w0 = key or -1)
8. Dispatch menu input:
   - KEY_UP (259): decrement _menu_selection (clamp to 0)
   - KEY_DOWN (258): increment _menu_selection (clamp to 7)
   - KEY_LEFT (260): for items 3-7, decrement setting value (level: clamp 1-22, toggles: set 0, noise: clamp 0-20)
   - KEY_RIGHT (261): for items 3-7, increment setting value (level: clamp 1-22, toggles: set 1, noise: clamp 0-20)
   - ENTER (10) or KEY_ENTER: for items 0-2, activate:
     - 0 (Start Game): set _game_state = 1
     - 1 (Help): set _game_state = 2
     - 2 (Quit): set _game_state = 0xFF (sentinel for exit)
   - 'q' (113) or ESC (27): set _game_state = 0xFF

**_help_frame():** One frame of the help state.
1. Load stdscr via GOT into x19
2. Call _werase(stdscr)
3. Render "-- CONTROLS --" title at row 2, col 10 via waddstr
4. Render 9 help lines starting at row 5, col 5, one per row (rows 5-13). Use _help_lines pointer table.
5. Render "Press any key to return" at row 16, col 5
6. Call _wrefresh(stdscr)
7. Call _poll_input -- if key != -1 (any key pressed), set _game_state = 0 (back to MENU)

**_draw_menu_number(w0 = number, w1 = row, w2 = col):** Helper to render a 1-2 digit number at a screen position. Uses the divide-by-10 stack buffer pattern from render.s _draw_number (or call _draw_number if it's globally accessible). Check render.s -- if _draw_number is .globl, call it. If not, implement a local version or make _draw_number global.

**Important conventions to follow:**
- A_REVERSE = 0x40000, constructed as `movz w_reg, #0x4, lsl #16` (same pattern as A_DIM in render.s)
- wattr_on/wattr_off take 3 args: (WINDOW*, attr_t, NULL) -- third arg x2=0
- All strings loaded via adrp+add for same-binary symbols
- stdscr loaded via GOT-indirect: `adrp x8, _stdscr@GOTPAGE` / `ldr x19, [x8, _stdscr@GOTPAGEOFF]` / `ldr x0, [x19]` to get WINDOW*
- Use KEY_ENTER constant: ncurses KEY_ENTER = 343 (0x157). Also check plain '\n' (10) for ENTER.
- Frame pacing: the existing wtimeout(16) from _init_input provides 16ms blocking in _poll_input/wgetch, giving ~60fps frame pacing. No additional sleep needed.

**Do NOT:**
- Use .ascii instead of .asciz (all strings must be null-terminated for waddstr)
- Forget to call wattr_off after wattr_on (would leave reverse video on for subsequent items)
- Use blocking input (wtimeout is already set to 16ms by _init_input)
- Put mutable state in __TEXT section (all mutable vars must be in __DATA,__data)

  <name>Task 2: Restructure main.s as a state machine with MENU/GAME/HELP outer loop</name>
  <files>asm/main.s</files>
  <action>
Rewrite _main in asm/main.s to introduce an outer state machine loop. The current linear flow (init -> game loop -> game over -> exit) becomes (init -> state loop -> cleanup -> exit).

**New _main structure:**

1. **Prologue** (keep existing): Save x19-x23 + x29/x30 in 64-byte frame.

2. **Initialization** (keep existing):
   - bl _initscr
   - bl _cbreak
   - bl _noecho
   - bl _init_colors
   - bl _init_input
   - **NEW:** Set _game_state = 0 (MENU) explicitly (already default from data.s, but be explicit for restarts)
   - **REMOVE:** The calls to _reset_board, _spawn_piece, _get_time_ms/_last_drop_time from here -- these move into the game start sequence triggered by the menu.

3. **State machine outer loop** (replaces the old Lgame_loop):

```
Lstate_loop:
    adrp    x8, _game_state@PAGE
    ldrb    w9, [x8, _game_state@PAGEOFF]

    // Dispatch by state
    cbz     w9, Lstate_menu         // 0 = MENU
    cmp     w9, #1
    b.eq    Lstate_game             // 1 = GAME
    cmp     w9, #2
    b.eq    Lstate_help             // 2 = HELP
    b       Lstate_exit             // 0xFF or unknown = EXIT

Lstate_menu:
    bl      _menu_frame
    b       Lstate_loop

Lstate_help:
    bl      _help_frame
    b       Lstate_loop

Lstate_game:
    // Check if game needs initialization (transition from menu)
    // Use a local flag or check if _game_over == 0 && _score == 0 && _lines_cleared == 0
    // Better approach: use a _game_initialized flag, or just check _game_state transition.
    //
    // Simplest: _menu_frame sets _game_state=1 on "Start Game". The first time we enter
    // Lstate_game, we need to initialize. Use a callee-saved register (e.g., w20) as
    // "game_running" flag: set to 0 before state loop, set to 1 after game init.
    // If w20==0 when entering GAME state, do init. If w20==1, do game frame.
    //
    // Implementation:
    //   Before Lstate_loop: mov w20, #0  (game not initialized)
    //   At Lstate_game: cbnz w20, Lgame_frame
    //   Game init: _reset_board, apply settings, _spawn_piece, _get_time_ms, mov w20, #1

    cbnz    w20, Lgame_frame

    // ---- GAME INITIALIZATION ----
    bl      _reset_board

    // Apply starting level: copy _starting_level to _level
    adrp    x8, _starting_level@PAGE
    ldr     w9, [x8, _starting_level@PAGEOFF]
    adrp    x8, _level@PAGE
    str     w9, [x8, _level@PAGEOFF]

    // Apply noise if _opt_noise > 0 (defer to Plan 04-02 -- just add the call site)
    // For now, leave a comment placeholder: bl _add_noise will be added in 04-02

    bl      _spawn_piece

    // Record initial gravity timestamp
    bl      _get_time_ms
    adrp    x8, _last_drop_time@PAGE
    str     x0, [x8, _last_drop_time@PAGEOFF]

    mov     w20, #1                 // mark game as initialized

Lgame_frame:
    // ---- EXISTING GAME LOOP BODY (one iteration) ----
    // This is the content of the old Lgame_loop, but ending with a state check
    // instead of looping back unconditionally.

    // 1. Poll input
    bl      _poll_input
    cmn     w0, #1
    b.eq    Lno_input2
    bl      _handle_input
Lno_input2:

    // 2. Check game over -> go to game over screen
    adrp    x8, _game_over@PAGE
    ldrb    w8, [x8, _game_over@PAGEOFF]
    cbnz    w8, Lgame_over_screen2

    // 2b. Check paused
    adrp    x8, _is_paused@PAGE
    ldrb    w8, [x8, _is_paused@PAGEOFF]
    cbnz    w8, Lskip_gravity2

    // 3. Gravity timer check
    bl      _get_time_ms
    mov     x19, x0                 // NOTE: x19 used for stdscr in menu.s but here in GAME state
                                    // it's safe because we're in a different state handler.
                                    // Actually, use x21-x23 to avoid confusion. But the existing
                                    // game loop used x19. Keep it consistent with the existing code.

    adrp    x8, _last_drop_time@PAGE
    ldr     x21, [x8, _last_drop_time@PAGEOFF]    // Use x21 instead of x20 (x20 = game_initialized flag)

    sub     x22, x19, x21          // elapsed = current - last_drop

    adrp    x8, _level@PAGE
    ldr     w23, [x8, _level@PAGEOFF]
    sub     w23, w23, #1

    adrp    x8, _gravity_delays@PAGE
    add     x8, x8, _gravity_delays@PAGEOFF
    ldrh    w9, [x8, x23, lsl #1]

    cmp     x22, x9
    b.lt    Lskip_gravity2

    bl      _soft_drop
    adrp    x8, _last_drop_time@PAGE
    str     x19, [x8, _last_drop_time@PAGEOFF]

Lskip_gravity2:
    bl      _render_frame

    // Check game over after render
    adrp    x8, _game_over@PAGE
    ldrb    w8, [x8, _game_over@PAGEOFF]
    cbz     w8, Lstate_loop         // not game over, continue state loop

    // Fall through to game over screen

Lgame_over_screen2:
    bl      _render_frame           // render with GAME OVER overlay

    // Switch to blocking input for game over screen
    adrp    x8, _stdscr@GOTPAGE
    ldr     x8, [x8, _stdscr@GOTPAGEOFF]
    ldr     x0, [x8]
    mov     w1, #-1
    bl      _wtimeout

Lwait_quit2:
    adrp    x8, _stdscr@GOTPAGE
    ldr     x8, [x8, _stdscr@GOTPAGEOFF]
    ldr     x0, [x8]
    bl      _wgetch
    cmp     w0, #113                // 'q'
    b.eq    Lreturn_to_menu
    cmp     w0, #27                 // ESC
    b.ne    Lwait_quit2

Lreturn_to_menu:
    // Transition back to menu
    adrp    x8, _game_state@PAGE
    strb    wzr, [x8, _game_state@PAGEOFF]    // _game_state = 0 (MENU)
    mov     w20, #0                             // reset game_initialized flag

    // Reset _menu_selection to 0
    adrp    x8, _menu_selection@PAGE
    strb    wzr, [x8, _menu_selection@PAGEOFF]

    // Restore non-blocking input for menu (wtimeout(16))
    adrp    x8, _stdscr@GOTPAGE
    ldr     x8, [x8, _stdscr@GOTPAGEOFF]
    ldr     x0, [x8]
    mov     w1, #16
    bl      _wtimeout

    b       Lstate_loop

Lstate_exit:
    // Cleanup and exit
    bl      _endwin
    mov     w0, #0
```

4. **Epilogue** (keep existing): Restore registers, ret.

- The old `_last_drop_time` was stored/loaded using x20 -- now it uses x21.
- Game over no longer calls _endwin + ret. Instead it transitions back to menu state.
- wtimeout restored to 16ms after game over screen (was set to -1 for blocking).
- The init sequence (_reset_board, _spawn_piece, _get_time_ms) only runs once per "Start Game" selection, not on program startup.

**IMPORTANT:** After the game over wait loop transitions back to menu, also clear the screen. Add `bl _werase` after restoring wtimeout but before `b Lstate_loop`:


**Do NOT:**
- Leave the old game init code at program start (must remove _reset_board, _spawn_piece, _get_time_ms from the linear init section)
- Forget to reset wtimeout from -1 back to 16 when returning to menu (would freeze the menu input)
- Forget to set w20=0 on return-to-menu (would skip game init on next "Start Game")
- Use x18 for anything (Apple reserved)
