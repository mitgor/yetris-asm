<task type="auto">
  <name>Task 1: Add mach_absolute_time frame timing to main.s game loop</name>
  <files>asm/main.s</files>
  <action>
Add frame timing instrumentation to the game loop in main.s. The goal is to measure actual game logic time per frame (excluding the wgetch blocking period).

**Prologue changes:**
- Expand stack frame from 64 to 96 bytes to save x24-x28 in addition to x19-x23
- Save x24, x25, x26, x27, x28 in the additional stack slots
- Update epilogue to restore x24-x28

**New data section (add at bottom of main.s before .subsections_via_symbols):**
```asm
.section __DATA,__data
.globl _frame_count
_frame_count: .quad 0
.globl _frame_time_sum
_frame_time_sum: .quad 0          // cumulative ticks
.globl _frame_time_min
_frame_time_min: .quad 0x7FFFFFFFFFFFFFFF  // start at max
.globl _frame_time_max
_frame_time_max: .quad 0
```

**Frame timing in Lgame_frame:**
After `_poll_input` returns and before game logic begins (right before the `_handle_input` call or the game_over check), insert:
```asm
    bl      _mach_absolute_time
    mov     x24, x0                 // x24 = frame_start_ticks
```

Before the `b Lstate_loop` at the end of the game frame (after `_render_frame` returns, after the game_over check), insert:
```asm
    bl      _mach_absolute_time
    sub     x25, x0, x24            // x25 = elapsed_ticks this frame

    // Update stats
    adrp    x8, _frame_count@PAGE
    add     x8, x8, _frame_count@PAGEOFF
    ldr     x9, [x8]
    add     x9, x9, #1
    str     x9, [x8]

    adrp    x8, _frame_time_sum@PAGE
    add     x8, x8, _frame_time_sum@PAGEOFF
    ldr     x9, [x8]
    add     x9, x9, x25
    str     x9, [x8]

    adrp    x8, _frame_time_min@PAGE
    add     x8, x8, _frame_time_min@PAGEOFF
    ldr     x9, [x8]
    cmp     x25, x9
    csel    x9, x25, x9, lo        // min = min(current, elapsed)
    str     x9, [x8]

    adrp    x8, _frame_time_max@PAGE
    add     x8, x8, _frame_time_max@PAGEOFF
    ldr     x9, [x8]
    cmp     x25, x9
    csel    x9, x25, x9, hi        // max = max(current, elapsed)
    str     x9, [x8]
```

**Print stats on exit in Lstate_exit:**
Before calling `_endwin`, print frame timing stats to stderr using write(2). Build a simple output string in a stack buffer. The approach:
1. Load _frame_count, _frame_time_sum, _frame_time_min, _frame_time_max
2. Convert ticks to microseconds: ticks * 125 / 3 / 1000
3. Compute average: sum_us / count
4. Format as: "Frames: NNNN  Min: NNNus  Max: NNNus  Avg: NNNus\n"
5. Use the existing divide-by-10 loop pattern (from render.s) for integer-to-ASCII
6. Write to stderr (fd=2) using: mov x0, #2; adrp x1, buffer; mov x2, length; mov x16, #4; svc #0x80

IMPORTANT: The timing should only be active during GAME state (not MENU or HELP). The mach_absolute_time calls are only in the Lgame_frame path, so this is naturally correct.

IMPORTANT: Measure AFTER _poll_input returns (excludes wgetch block time) and AFTER _render_frame returns. This captures: input dispatch + game_over check + pause check + gravity logic + full render. Per the research pitfall: "Start timer AFTER _poll_input returns, stop BEFORE wrefresh" -- however since _render_frame includes wrefresh internally, we measure the render too (which is the actual work, not blocking I/O).

Keep the output formatting simple. A fixed-format string with embedded numbers is sufficient. Do not use printf/fprintf (would require variadic calling convention). Use raw write syscall to stderr.

<task type="auto">
  <name>Task 2: Binary size optimization and Instruments profiling</name>
  <files>Makefile</files>
  <action>
**Makefile changes:**

1. Add `-dead_strip` to the ld command:
```makefile
$(ASM_EXE): $(ASM_OBJECTS) | $(ASM_BIN_DIR)
	# Linking assembly binary...
	$(MUTE)ld -o $@ $(ASM_OBJECTS) -lncurses -lSystem \
		-syslibroot $(SDK_PATH) -arch arm64 \
		-dead_strip
```

2. Add a new `asm-strip` target that creates a stripped copy:
```makefile
asm-strip: asm
	# Creating stripped binary...
	$(MUTE)cp $(ASM_EXE) $(ASM_EXE)-stripped
	$(MUTE)strip -x $(ASM_EXE)-stripped
	@echo "Stripped binary: $(ASM_EXE)-stripped"
	@ls -la $(ASM_EXE) $(ASM_EXE)-stripped

asm-profile: asm
	# Recording 10-second Time Profiler trace...
	xcrun xctrace record \
		--template 'Time Profiler' \
		--time-limit 10s \
		--output yetris-profile.trace \
		--launch -- ./$(ASM_EXE)
```

3. Add `asm-strip asm-profile` to the .PHONY line:
```makefile
.PHONY: asm asm-clean asm-run asm-strip asm-profile
```
