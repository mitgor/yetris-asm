**Step 2: Add NEON mask data to data.s**

Add a 16-byte mask in the __TEXT,__const section of data.s (near the other read-only data). This mask has bytes 0-9 as 0x00 and bytes 10-15 as 0xFF:

```asm
// NEON mask for line detection: padding bytes 10-15 set to 0xFF
// so uminv across 16 bytes only tests the 10 real board cells
.globl _neon_row_mask
.p2align 4                          // 16-byte aligned for NEON load
_neon_row_mask:
    .byte 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00, 0x00
    .byte 0x00, 0x00, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF, 0xFF
```

Place this AFTER the existing const data (after _piece_spawn_y or similar) but still within the __TEXT,__const section.

**Step 3: Replace scalar row check in _clear_lines (board.s)**

In _clear_lines, replace the scalar Lclear_check_col loop (the inner loop that checks if a row is full by iterating 10 bytes) with the NEON pattern:

Replace this block:
```asm
    // Check if row is full (all 10 cells non-zero)
    mov     w22, #0                 // col = 0
    mov     w10, #10
    mul     w10, w20, w10           // row * 10 = row offset
    uxtw    x10, w10
Lclear_check_col:
    ldrb    w11, [x19, x10]        // board[row*10 + col]
    cbz     w11, Lclear_not_full    // if any cell is 0, row not full
    add     x10, x10, #1
    add     w22, w22, #1
    cmp     w22, #10
    b.lt    Lclear_check_col
```

With:
```asm
    // NEON full-row check: vectorized min across 16 bytes
    mov     w10, #10
    mul     w10, w20, w10           // row * 10 = row offset
    uxtw    x10, w10
    add     x10, x19, x10          // x10 = &board[row*10]

    ld1     {v0.16b}, [x10]        // load 16 bytes (10 data + 6 junk)
    adrp    x11, _neon_row_mask@PAGE
    add     x11, x11, _neon_row_mask@PAGEOFF
    ldr     q1, [x11]              // load mask (0x00*10 + 0xFF*6)
    orr     v0.16b, v0.16b, v1.16b // force padding bytes to non-zero
    uminv   b2, v0.16b             // unsigned min across all 16 bytes
    umov    w11, v2.b[0]           // extract scalar minimum
    cbz     w11, Lclear_not_full   // if min==0, at least one cell empty
```

IMPORTANT: The ld1 loads 16 bytes starting at &board[row*10]. For rows 0-18 this reads 6 bytes into the next row (which is fine -- those bytes get masked to 0xFF). For row 19 (the last row, starting at offset 190), loading 16 bytes reads up to offset 206, which is 6 bytes past the end of the 200-byte _board array. To be safe, add 6 bytes of padding AFTER _board in data.s:

In data.s, after _board:
```asm
_board:
    .space 200                      // 20 rows x 10 cols
    .space 6                        // NEON padding: ld1 reads 16 bytes from last row
```

Or better: change the _board allocation from `.space 200` to `.space 208` (adding 8 bytes of padding for alignment). This is invisible to all other code since nothing reads past offset 199.

**Step 2: Pack game state into x28 in main.s**

Currently main.s reads game_over and is_paused from memory via adrp+ldrb on every frame iteration. Pack these into x28 bitfield:

- Bit 0: game_over
- Bit 1: is_paused
- Bit 2: game_initialized (currently w20)

**Changes to main.s:**

1. In prologue: save x28 (already saved if stack frame was expanded in Plan 01). Initialize x28 to 0.

2. Replace the `mov w20, #0` (game_initialized = 0) with `bic x28, x28, #4` (clear bit 2). Replace `mov w20, #1` with `orr x28, x28, #4` (set bit 2). Replace `cbnz w20, Lgame_frame` with `tst x28, #4` / `b.ne Lgame_frame`.

3. Replace the game_over memory reads in Lgame_frame:
```asm
// BEFORE:
    adrp    x8, _game_over@PAGE
    ldrb    w8, [x8, _game_over@PAGEOFF]
    cbnz    w8, Lgame_over_screen

// AFTER:
    tst     x28, #1                 // test bit 0 (game_over)
    b.ne    Lgame_over_screen
```

4. Replace the is_paused memory read:
```asm
// BEFORE:
    adrp    x8, _is_paused@PAGE
    ldrb    w8, [x8, _is_paused@PAGEOFF]
    cbnz    w8, Lskip_gravity

// AFTER:
    tst     x28, #2                 // test bit 1 (is_paused)
    b.ne    Lskip_gravity
```

5. After `_handle_input` returns (which may set game_over or is_paused in memory), SYNC from memory to x28:
```asm
    // Sync memory flags to x28 after input handling
    adrp    x8, _game_over@PAGE
    ldrb    w9, [x8, _game_over@PAGEOFF]
    bic     x28, x28, #1
    orr     x28, x28, x9           // bit 0 = game_over

    adrp    x8, _is_paused@PAGE
    ldrb    w9, [x8, _is_paused@PAGEOFF]
    bic     x28, x28, #2
    orr     x28, x28, x9, lsl #1   // bit 1 = is_paused
```

6. Similarly, after `_soft_drop` returns (which may trigger game_over via lock+spawn), sync game_over:
```asm
    adrp    x8, _game_over@PAGE
    ldrb    w9, [x8, _game_over@PAGEOFF]
    bic     x28, x28, #1
    orr     x28, x28, x9           // sync game_over after gravity
```

7. After the render frame, the second game_over check now uses the register:
```asm
    tst     x28, #1
    b.ne    Lgame_over_screen
```

8. On Lreturn_to_menu, reset x28:
```asm
    mov     x28, #0                 // clear all packed state
```

CRITICAL: The memory globals (_game_over, _is_paused) are still the source of truth -- they are written by input.s, board.s, piece.s. Register packing only caches them in main.s game loop to avoid redundant adrp+ldrb on each frame iteration. The sync points after _handle_input and _soft_drop reload from memory into x28.
