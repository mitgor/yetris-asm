<!-- Phase 1 ABI patterns -->
- Prologue: stp x20,x19,[sp,#-32]! / stp x29,x30,[sp,#16] / add x29,sp,#16 (or larger frame if more callee-saved regs needed)
- Local data access: adrp xN, label@PAGE / add xN, xN, label@PAGEOFF

  <action>
Create asm/board.s with the following functions. All functions operate on the global state variables in data.s.

**1. _is_piece_valid(w0=type, w1=rotation, w2=px, w3=py) -> w0=1 (valid) or w0=0 (invalid)**

The core collision detection function. For each cell (r,c) in the piece's 5x5 grid:
- Load piece_data[type*100 + rotation*25 + r*5 + c]
- If cell == 0: skip (empty cell)
- Compute board_x = px + c, board_y = py + r
- If board_x < 0 or board_x >= 10: return 0 (wall collision)
- If board_y >= 20: return 0 (floor collision)
- If board_y < 0: continue (above board, not yet visible -- this is valid)
- If board[board_y * 10 + board_x] != 0: return 0 (block collision)
- After all 25 cells checked without collision: return 1

Implementation details:
- Save all 4 arguments in callee-saved registers (x19-x22) since they are needed across the 25-iteration loop
- Load _piece_data base address via adrp+add (read-only, same binary)
- Load _board base address via adrp+add (mutable, same binary)
- Use w23 for row counter (0-4), w24 for col counter (0-4)
- Inner loop: 5 columns per row, 5 rows = 25 iterations total
- Use SIGNED comparisons for board_x and board_y (pieces can be at negative positions during spawn)
- px and py are SIGNED (loaded with ldrsh from .hword in caller, passed as signed w2/w3)

Prologue must save x19-x24 + x29/x30 = 48 bytes minimum (round to 64 for alignment). Use a larger frame: stp x28,x27,[sp,#-64]! / stp x26,x25,[sp,#16] / stp x24,x23,[sp,#32] / stp x22,x21,[sp,#48] etc., then save x29/x30 and set up frame pointer.

Actually, use a cleaner approach: save only what you need. 6 callee-saved regs (x19-x24) + x29/x30 = 8 regs = 64 bytes:
```
stp x29, x30, [sp, #-64]!
stp x19, x20, [sp, #16]
stp x21, x22, [sp, #32]
stp x23, x24, [sp, #48]
mov x29, sp
```

**2. _lock_piece() -> w0 = number of lines cleared (0-4)**

Locks the current piece into the board and handles scoring/line clearing:
1. Load _piece_type, _piece_rotation, _piece_x, _piece_y from data.s globals
2. For each cell in the piece's 5x5 grid that is non-zero:
   - Compute board position: bx = piece_x + c, by = piece_y + r
   - If by >= 0 and by < 20 and bx >= 0 and bx < 10: set board[by*10 + bx] = piece_type + 1 (store 1-7 for color, not 0)
3. Add 10 to _score (lock bonus)
4. Call _clear_lines to detect and remove full rows
5. Return the number of lines cleared in w0

**3. _clear_lines() -> w0 = number of lines cleared (0-4)**

Scan the board bottom-to-top for full rows and remove them:
1. Initialize lines_cleared_count = 0
2. For row from 19 down to 0:
   - Check if all 10 cells in the row are non-zero (full row)
   - If full: increment lines_cleared_count, shift all rows above down by 1 (copy row j-1 to row j for j from current row down to 1), fill row 0 with zeros, and re-check the current row (it now has new content)
3. If lines_cleared_count > 0:
   - Load _score_table[lines_cleared_count - 1] and add to _score
   - Add lines_cleared_count to _lines_cleared
   - Recompute _level: scan _level_thresholds to find the highest level whose threshold <= _lines_cleared. Level = index + 1.
4. Return lines_cleared_count in w0

CRITICAL: Scan bottom-to-top. When a full line is found, shift rows above down, clear top row, and re-check same index (as per research Pitfall 6).

**4. _reset_board()**

Zero all 200 bytes of _board. Simple memset-equivalent loop. Also reset _score to 0, _level to 1, _lines_cleared to 0, _game_over to 0.

- All 4 functions with .globl and .p2align 2
- .subsections_via_symbols at end
- No data section needed -- all data is in data.s

<task type="auto">
  <name>Task 2: Create piece.s with movement, SRS rotation, drops, spawning, and game over</name>
  <files>asm/piece.s</files>
  <action>
Create asm/piece.s with the following functions. All functions read/write the global state variables in data.s and call board.s functions.

**1. _try_move(w0=dx, w1=dy) -> w0=1 (moved) or w0=0 (blocked)**

Attempt to move the current piece by (dx, dy):
1. Load current _piece_type, _piece_rotation, _piece_x, _piece_y
2. Compute new_x = piece_x + dx, new_y = piece_y + dy
3. Call _is_piece_valid(piece_type, piece_rotation, new_x, new_y)
4. If valid: update _piece_x = new_x, _piece_y = new_y, return 1
5. If invalid: return 0 (position unchanged)

dx and dy are signed. For left move: dx=-1, dy=0. Right: dx=+1, dy=0. Down: dx=0, dy=+1.

**2. _try_rotate(w0=direction) -> w0=1 (rotated) or w0=0 (blocked)**

Attempt SRS rotation with wall kicks:
1. Load current _piece_type, _piece_rotation, _piece_x, _piece_y
2. Compute new_rotation = (piece_rotation + direction) & 3 (direction is +1 for CW, +3 for CCW -- using +3 instead of -1 to avoid negative modulo)
3. Try basic rotation: call _is_piece_valid(type, new_rotation, piece_x, piece_y). If valid, update _piece_rotation and return 1.
4. If basic rotation fails, determine kick table:
   - If piece_type == 1 (I-piece): use _srs_kicks_i
   - Else: use _srs_kicks_jlstz
5. Determine direction index: if direction == 1 (CW): dir_idx = 0. If direction == 3 (CCW): dir_idx = 1.
6. For test = 1 to 4 (test 0 was the basic rotation already tried):
   - Load dx = kick_table[dir_idx*40 + piece_rotation*10 + test*2 + 0] using ldrsb (signed!)
   - Load dy = kick_table[dir_idx*40 + piece_rotation*10 + test*2 + 1] using ldrsb (signed!)
   - NEGATE dy for board coordinates: kick_y = -dy (SRS uses math convention +Y=up, board uses +Y=down)
   - Call _is_piece_valid(type, new_rotation, piece_x + dx, piece_y + kick_y)
   - If valid: update _piece_x += dx, _piece_y += kick_y, _piece_rotation = new_rotation, return 1
7. All 5 tests failed: return 0 (rotation cancelled)

CRITICAL: The SRS Y-axis inversion (negate dy) is essential. Without it, kicks will push pieces through walls instead of away from them. The C++ reference does this in Piece::moveBy with `this->y -= dy`.

**3. _hard_drop() -> void**

Instantly drop the current piece to its lowest valid position and lock:
1. Load current piece state
2. Loop: try_y = piece_y + 1. Call _is_piece_valid(type, rotation, piece_x, try_y). If valid, piece_y = try_y, repeat. If invalid, stop.
3. Update _piece_y with the final valid position
4. Call _lock_piece()
5. Call _spawn_piece()

**4. _soft_drop() -> w0=1 (moved down) or w0=0 (locked)**

Move piece down by one. If blocked, lock and spawn next:
1. Call _try_move(0, 1)
2. If move succeeded: return 1
3. If blocked: call _lock_piece(), call _spawn_piece(), return 0

**5. _spawn_piece() -> void**

Spawn the next piece from the 7-bag:
1. Call _next_piece() to get piece type (0-6) in w0
2. Store to _piece_type
3. Set _piece_rotation = 0
4. Load spawn position: _piece_x = _piece_spawn_x[type], _piece_y = _piece_spawn_y[type] (use ldrsb -- spawn_y can be negative)
5. Call _check_game_over()

**6. _check_game_over() -> void**

Check if the just-spawned piece collides:
1. Load current _piece_type, _piece_rotation, _piece_x, _piece_y
2. Call _is_piece_valid(type, rotation, piece_x, piece_y)
3. If invalid: set _game_over = 1

**File structure:**
- Header comment block
- .section __TEXT,__text,regular,pure_instructions
- All 6 functions with .globl and .p2align 2
- .subsections_via_symbols at end
- No data section needed

**Register allocation strategy for complex functions (try_rotate):**
Use callee-saved registers aggressively:
- x19: piece_type
- x20: piece_rotation (original)
- x21: piece_x
- x22: piece_y
- x23: new_rotation
- x24: kick table base address
- x25: test counter
- x26: direction index
Save all used callee-saved regs in prologue, restore in epilogue.
