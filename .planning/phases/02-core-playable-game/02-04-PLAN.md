Completely rewrite asm/main.s. Replace the Phase 1 hello-world program with the full Tetris game loop. Preserve the Darwin ABI header documentation comment block from Phase 1 (it documents conventions used throughout the project), but replace all code.

**Program structure:**

```
_main:
    prologue (save callee-saved registers)

    // === INITIALIZATION ===
    bl _initscr              // init ncurses
    bl _cbreak               // disable line buffering
    bl _noecho               // suppress echo
    bl _init_colors          // setup 7 color pairs + hide cursor
    bl _init_input           // keypad + wtimeout(16) + noecho + cbreak
    bl _reset_board          // zero board, reset score/level/lines
    bl _spawn_piece          // get first piece from 7-bag

    // Record initial gravity timestamp
    bl _get_time_ms
    // Store result (x0) to _last_drop_time
    adrp x8, _last_drop_time@PAGE
    str  x0, [x8, _last_drop_time@PAGEOFF]

    // === GAME LOOP ===
game_loop:
    // 1. Poll input
    bl _poll_input           // w0 = key or -1
    cmn w0, #1               // compare with -1 (ERR)
    b.eq no_input            // skip if no key pressed
    bl _handle_input         // dispatch key to game action
no_input:

    // 2. Check game over (handle_input may have set it via 'q' or spawn collision)
    adrp x8, _game_over@PAGE
    ldrb w8, [x8, _game_over@PAGEOFF]
    cbnz w8, game_over_screen

    // 3. Gravity timer check
    bl _get_time_ms          // x0 = current time in ms
    mov x19, x0              // save current time in callee-saved reg

    // Load last_drop_time
    adrp x8, _last_drop_time@PAGE
    ldr  x20, [x8, _last_drop_time@PAGEOFF]

    // Compute elapsed = current - last_drop
    sub  x21, x19, x20       // x21 = elapsed ms

    // Load gravity delay for current level
    adrp x8, _level@PAGE
    ldr  w22, [x8, _level@PAGEOFF]     // w22 = level (1-22)
    sub  w22, w22, #1                    // index = level - 1

    adrp x8, _gravity_delays@PAGE
    add  x8, x8, _gravity_delays@PAGEOFF
    ldrh w23, [x8, x22, lsl #1]        // w23 = delay_ms for current level

    // If elapsed >= delay: apply gravity
    cmp  x21, x23
    b.lt skip_gravity

    // Apply gravity: try to move piece down
    bl   _soft_drop           // moves down or locks+spawns

    // Reset gravity timer
    adrp x8, _last_drop_time@PAGE
    str  x19, [x8, _last_drop_time@PAGEOFF]   // last_drop = current_time

skip_gravity:

    // 4. Render frame
    bl _render_frame

    // 5. Loop unless game over
    adrp x8, _game_over@PAGE
    ldrb w8, [x8, _game_over@PAGEOFF]
    cbz  w8, game_loop

    // === GAME OVER ===
game_over_screen:
    bl _render_frame          // render one more time to show GAME OVER

    // Wait for 'q' key to exit (switch to blocking input)
    // Load stdscr, set timeout to -1 (blocking)
    adrp x8, _stdscr@GOTPAGE
    ldr  x8, [x8, _stdscr@GOTPAGEOFF]
    ldr  x0, [x8]
    mov  w1, #-1
    bl   _wtimeout            // blocking mode

wait_quit:
    adrp x8, _stdscr@GOTPAGE
    ldr  x8, [x8, _stdscr@GOTPAGEOFF]
    ldr  x0, [x8]
    bl   _wgetch
    cmp  w0, #113             // 'q'
    b.ne wait_quit

    // === CLEANUP ===
    bl _endwin                // restore terminal

    mov w0, #0                // return 0
    epilogue (restore callee-saved registers)
    ret
```

**Register allocation in game loop:**
- x19: current time (used for gravity comparison)
- x20: last_drop_time
- x21: elapsed time
- x22: level index (level - 1)
- x23: gravity delay for current level

Prologue must save x19-x23 + x29/x30. That is 5 callee-saved + 2 = 7 registers = need 64 bytes (round up from 56 for 16-byte alignment).

Frame setup:
```asm
stp x29, x30, [sp, #-64]!
stp x19, x20, [sp, #16]
stp x21, x22, [sp, #32]
str x23, [sp, #48]
mov x29, sp
```

Epilogue:
```asm
ldr x23, [sp, #48]
ldp x21, x22, [sp, #32]
ldp x19, x20, [sp, #16]
ldp x29, x30, [sp], #64
ret
```

**Keep the ABI documentation header from Phase 1** (the comment block at the top of main.s documenting all 9 Darwin ABI rules). Update the file description line to say "Entry point, game initialization, main game loop" instead of "Entry point, ncurses initialization, hello world".

**Remove the old hello_str and the old code** (printw, wgetch for keypress wait, etc). The .section __TEXT,__cstring section can be removed entirely (no strings needed in main.s -- all string data is in render.s).

**Key implementation details:**
- The gravity delay for level 22 is 0ms, meaning the piece should drop every frame. The `cmp x21, x23` / `b.lt skip_gravity` handles this: when delay is 0, elapsed (which is always >= 0) is always >= 0, so gravity fires every frame.
- After game over, switch wtimeout to -1 (blocking) so the program waits for the user to press 'q'. This prevents a busy-wait loop burning CPU.
- The game_over check happens TWICE per loop: once after input (to catch 'q' press or spawn collision from hard_drop), and once after gravity (to catch spawn collision from gravity-triggered lock). This ensures the game stops immediately on game over.

**Anti-patterns to avoid:**
- Do NOT use usleep or napms for frame timing. The 16ms wtimeout provides natural frame pacing via wgetch's blocking period.
- Do NOT call wrefresh outside of _render_frame. The game loop calls _render_frame once per iteration.
- Do NOT forget to initialize _last_drop_time before the loop. Without it, the first gravity tick will fire immediately (elapsed = current_time - 0 = huge number).
