From asm/main.s (Phase 1 patterns):
- Prologue: stp x20,x19,[sp,#-32]! / stp x29,x30,[sp,#16] / add x29,sp,#16
- Epilogue: ldp x29,x30,[sp,#16] / ldp x20,x19,[sp],#32 / ret
- Local data: adrp xN, label@PAGE / add xN, xN, label@PAGEOFF
- External globals: adrp xN, sym@GOTPAGE / ldr xN, [xN, sym@GOTPAGEOFF] / ldr xN, [xN]
- Sections: .section __TEXT,__text for code; .section __TEXT,__const for read-only data; .section __DATA,__data for mutable state
- Every file ends with: .subsections_via_symbols
- Every function: .globl _funcname / .p2align 2
- NEVER use x18. Caller-saved: x0-x15. Callee-saved: x19-x28.

From 02-RESEARCH.md (data layout contracts):
- Piece data: 5x5 byte grid format, 7 types x 4 rotations x 5 rows x 5 cols = 700 bytes
  - Index: type*100 + rotation*25 + row*5 + col
  - Values: 0=empty, 1=block, 2=pivot
  - Piece order: O=0, I=1, L=2, J=3, S=4, Z=5, T=6

- SRS kick tables: 2 tables (JLSTZ and I-piece)
  - Each: 2 directions (CW=0, CCW=1) x 4 start rotations x 5 tests x 2 axes = 80 signed bytes per table
  - Index: direction*40 + start_rotation*10 + test*2 + axis (0=dx, 1=dy)
  - Values are SIGNED bytes -- must use ldrsb to load

- Gravity delays: 22-entry .hword table (ms per level)
  - Values: 1000,900,800,700,600,500,450,400,350,300,250,200,150,120,100,80,60,40,30,20,10,0

- Score table: 4-entry .word table: 100, 300, 500, 800
  - Index: (lines_cleared - 1) * 4

- Level thresholds: 22-entry .hword table
  - Values: 5,10,15,20,25,30,40,50,60,70,100,120,140,160,180,210,240,280,310,350,400,450

- Color pairs: 7-entry byte table mapping piece type to ncurses color pair number
  - O=1(yellow), I=2(cyan), L=3(white), J=4(blue), S=5(green), Z=6(red), T=7(magenta)

- Game state variables (mutable, in __DATA,__data):
  - _board: .space 200, 0 (10x20 byte array, row-major)
  - _piece_type: .byte 0 (current piece type 0-6)
  - _piece_rotation: .byte 0 (0-3)
  - _piece_x: .hword 0 (signed 16-bit)
  - _piece_y: .hword 0 (signed 16-bit)
  - _score: .word 0 (unsigned 32-bit)
  - _level: .word 1 (starts at 1)
  - _lines_cleared: .word 0
  - _game_over: .byte 0
  - _bag: .space 7, 0
  - _bag_index: .byte 7 (starts at 7 to trigger initial refill)
  - _last_drop_time: .quad 0 (64-bit ms timestamp)

Create asm/data.s containing ALL game data tables in read-only sections and ALL mutable game state in the data section. This file is the single source of truth for all game constants and state variables.

**Read-only data (.section __TEXT,__const):**

1. **_piece_data** (700 bytes): 7 tetrominoes in 5x5 grid format with 4 rotations each. Use the C++ reference PieceDefinitions.cpp piece order: O=0, I=1, L=2, J=3, S=4, Z=5, T=6. Values: 0=empty, 1=block, 2=pivot. The 5x5 grid centers the piece -- each rotation state occupies exactly 25 bytes.

   Transcribe ALL 28 rotation states (7 types x 4 rotations) from the tetris.wiki SRS canonical rotation states. For each piece type, define rotations 0 (spawn), 1 (CW from 0 / R), 2 (180 / 2), 3 (CCW from 0 / L). Use comments to label each piece and rotation clearly.

   Critical: The I-piece rotation states must match SRS exactly. The I-piece spawns horizontal in a 5x5 grid row 2 (0-indexed row 1). The O-piece all 4 rotations are identical (2x2 block in the top-left of a 5x5 grid centered).

2. **_srs_kicks_jlstz** (80 signed bytes): SRS wall kick offset table for J, L, S, T, Z pieces. Layout: [CW][4 rotations][5 tests][2 axes] then [CCW][4 rotations][5 tests][2 axes]. Each value is a signed byte (dx or dy). The kick test 0 is always (0,0) for all rotations.

   Use the EXACT values from the SRS specification (tetris.wiki):
   - CW 0->R: (0,0),(-1,0),(-1,+1),(0,-2),(-1,-2)
   - CW R->2: (0,0),(+1,0),(+1,-1),(0,+2),(+1,+2)
   - CW 2->L: (0,0),(+1,0),(+1,+1),(0,-2),(+1,-2)
   - CW L->0: (0,0),(-1,0),(-1,-1),(0,+2),(-1,+2)
   - CCW is the reverse: 0->L, L->2, 2->R, R->0 with negated offsets

   IMPORTANT: These offsets use math convention (positive Y = up). When applying to board coordinates (positive Y = down), the Y component must be NEGATED. This negation happens in piece.s (plan 02), NOT here. Store the raw SRS values.

3. **_srs_kicks_i** (80 signed bytes): Same layout as JLSTZ but with I-piece specific offsets from SRS specification.

4. **_gravity_delays** (44 bytes): 22-entry .hword table. Level 1 = 1000ms down to level 22 = 0ms. Values: 1000,900,800,700,600,500,450,400,350,300,250,200,150,120,100,80,60,40,30,20,10,0.

5. **_score_table** (16 bytes): 4-entry .word table: 100, 300, 500, 800 (for 1,2,3,4 lines cleared).

6. **_level_thresholds** (44 bytes): 22-entry .hword table of cumulative line thresholds for each level. Values: 5,10,15,20,25,30,40,50,60,70,100,120,140,160,180,210,240,280,310,350,400,450.

7. **_color_pairs** (7 bytes): Maps piece type index (0-6) to ncurses color pair number (1-7). O->1, I->2, L->3, J->4, S->5, Z->6, T->7.

8. **_piece_spawn_x** and **_piece_spawn_y** (7 signed bytes each): Spawn position for each piece type. All pieces spawn at x=2 (to center the 5x5 grid on the 10-wide board: 2 + 5 = 7, leaving columns 0-1 and 7-9). Spawn y = -2 for most pieces (above the board so the lowest blocks appear at row 0). Adjust per piece: O spawns at y=-1 (only 2 rows tall), I spawns at y=-2.

**Mutable state (.section __DATA,__data):**

All game state variables as specified in the interfaces block above. Use .globl for every label. Include clear comments.

**File structure:**
- Start with a header comment block (similar style to main.s)
- .section __TEXT,__const for all read-only tables
- .section __DATA,__data for all mutable state
- End with .subsections_via_symbols

**Anti-patterns to avoid:**
- Do NOT use the compact 4-cell representation. Use 5x5 grids to match the C++ reference collision detection algorithm exactly. This costs 700 bytes but eliminates a class of bugs.
- Do NOT put data tables in __DATA -- they are read-only constants and belong in __TEXT,__const.
- Do NOT forget .globl on every exported label.

Create two utility source files that provide timer and random number services to the game.

**asm/timer.s -- gettimeofday wrapper:**

Create a single function `_get_time_ms` that:
1. Allocates 16 bytes on stack for struct timeval (tv_sec: 8 bytes at offset 0, tv_usec: 4 bytes at offset 8)
2. Calls `bl _gettimeofday` with x0 = stack pointer, x1 = 0 (NULL timezone)
3. Loads tv_sec (ldr x8, [sp]) and tv_usec (ldrsw x9, [sp, #8] -- sign-extend 32 to 64)
4. Computes result = tv_sec * 1000 + tv_usec / 1000
5. Returns result in x0

Use the exact pattern from 02-RESEARCH.md Code Examples "Timer Using gettimeofday" section. The function needs a 32-byte stack frame (16 for timeval + 16 for saved fp/lr).

Prologue/epilogue must follow the Phase 1 pattern. Mark with .globl _get_time_ms / .p2align 2.

**asm/random.s -- 7-bag random piece generator:**

Create two functions:

1. `_shuffle_bag`: Fills _bag[0..6] with a Fisher-Yates shuffle of indices 0-6 and resets _bag_index to 0.
   - Initialize bag with sequential values 0,1,2,...,6
   - For i from 6 down to 1: call `bl _arc4random_uniform` with w0 = i+1, get random j in [0,i], swap bag[i] and bag[j]
   - Reset _bag_index to 0
   - Use callee-saved registers (x19-x22) for: bag address, loop counter, random result
   - Load _bag address via adrp+add (@PAGE/@PAGEOFF since it is in same binary)
   - Load _bag_index address via adrp+add

2. `_next_piece`: Returns the next piece type (0-6) in w0. If _bag_index >= 7, calls _shuffle_bag first to refill. Then reads bag[bag_index], increments bag_index, returns the value.
   - Load _bag_index, check if >= 7
   - If so, bl _shuffle_bag (which resets index to 0)
   - Load bag[bag_index] via ldrb
   - Increment bag_index, store back
   - Return piece type in w0

Both functions need proper prologue/epilogue (save/restore x19-x22 + x29/x30). End file with .subsections_via_symbols.

- Do NOT forget to save/restore callee-saved registers used inside each function.
