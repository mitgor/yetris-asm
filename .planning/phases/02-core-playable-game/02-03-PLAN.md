<task type="auto">
  <name>Task 1: Create render.s with board, piece, score panel, and color rendering</name>
  <files>asm/render.s</files>
  <action>
Create asm/render.s with all visual rendering functions. Use `stdscr` for all drawing (no separate ncurses windows in Phase 2, per research recommendation). All rendering uses wmove+waddch to place characters at computed screen coordinates.

**Screen layout (using stdscr directly):**
```
Column:  0         10        20        30
Row 0:   +--------------------+  Score
Row 1:   |                    |  ------
Row 2:   |   (board area)     |  12345
Row 3:   |   10 cols x 2 ch   |
Row 4:   |   = 20 chars wide  |  Level
...      |                    |  ------
Row 19:  |                    |  5
Row 20:  |                    |
Row 21:  +--------------------+  Lines
                                 ------
                                 42
```

Board area: rows 1-20, columns 1-20 (each board cell = 2 chars wide: "[]")
- Board top-left on screen: row=1, col=1
- Board cell (board_row, board_col) maps to screen (row+1, col*2+1)
- Borders: row 0 and row 21 are "+" and "-", columns 0 and 21 are "|"

Score panel: starts at column 24, rows 1-12
- "Score" label at (1, 24), score value at (2, 24)
- "Level" label at (4, 24), level value at (5, 24)
- "Lines" label at (7, 24), lines value at (8, 24)

**1. _init_colors() -> void**

Initialize ncurses color support:
1. Call `bl _start_color`
2. Call `bl _init_pair` for each of the 7 color pairs:
   - Pair 1: COLOR_YELLOW(3) on COLOR_BLACK(0) -- O piece
   - Pair 2: COLOR_CYAN(6) on COLOR_BLACK(0) -- I piece
   - Pair 3: COLOR_WHITE(7) on COLOR_BLACK(0) -- L piece (orange substitute)
   - Pair 4: COLOR_BLUE(4) on COLOR_BLACK(0) -- J piece
   - Pair 5: COLOR_GREEN(2) on COLOR_BLACK(0) -- S piece
   - Pair 6: COLOR_RED(1) on COLOR_BLACK(0) -- Z piece
   - Pair 7: COLOR_MAGENTA(5) on COLOR_BLACK(0) -- T piece
3. Call `bl _curs_set` with w0=0 to hide cursor

Load stdscr GOT pointer into x19 at the start (will be reused by all render functions).

**2. _draw_board() -> void**

Draw the board borders and all locked blocks:
1. Load stdscr via GOT into a callee-saved register
2. Call `bl _werase` to clear the screen first (pass stdscr as WINDOW*)
3. Draw top border: wmove to (0, 0), draw "+" then 20 "-" chars then "+"
4. Draw bottom border: same pattern at row 21
5. For each board row (0-19):
   - Draw left border "|" at screen (row+1, 0)
   - For each board col (0-9):
     - Load board[row*10 + col]
     - If 0 (empty): draw "  " (two spaces) at screen (row+1, col*2+1)
     - If non-zero (value v, 1-7): set color pair v (wattr_on with COLOR_PAIR(v) = v<<8), draw "[]", then wattr_off
   - Draw right border "|" at screen (row+1, 21)

Use callee-saved registers for: stdscr ptr (x19), board base address (x20), row counter (x21), col counter (x22).

**3. _draw_piece() -> void**

Draw the current falling piece on top of the board:
1. Load _piece_type, _piece_rotation, _piece_x, _piece_y
2. Compute color pair: load _color_pairs[piece_type]
3. For each cell (r, c) in the piece's 5x5 grid:
   - Load piece_data[type*100 + rotation*25 + r*5 + c]
   - If 0: skip
   - Compute screen_y = piece_y + r + 1 (add 1 for top border)
   - Compute screen_x = (piece_x + c) * 2 + 1 (add 1 for left border)
   - If screen_y < 1 or screen_y > 20: skip (above or below visible board)
   - If screen_x < 1 or screen_x > 20: skip (outside visible board)
   - Set color, wmove to (screen_y, screen_x), draw "[]", unset color

**4. _draw_score_panel() -> void**

Draw score, level, and lines labels and values:
1. Draw "Score" label at (1, 24) using wmove + waddch for each character
2. Load _score, convert to ASCII digits, draw at (2, 24) right-justified in 8 chars
3. Draw "Level" label at (4, 24)
4. Load _level, convert to ASCII digits, draw at (5, 24)
5. Draw "Lines" label at (7, 24)
6. Load _lines_cleared, convert to ASCII digits, draw at (8, 24)

For integer-to-ASCII conversion, use inline digit extraction: divide by 10 repeatedly, store digits right-to-left into a small stack buffer (8 bytes), then wmove+waddch each character. Do NOT call mvwprintw (variadic pitfall).

Helper approach: allocate 16 bytes on stack for digit buffer. Divide w0 by 10, store remainder + 0x30 (ASCII '0'). Repeat until quotient is 0. Then draw digits left-to-right from the buffer. Pad with spaces on the left to fill 8 characters.

**5. _draw_game_over() -> void**

Draw "GAME OVER" text centered on the board:
1. wmove to approximately (10, 5) -- center of board area on screen
2. Draw "GAME OVER" character by character with waddch
3. wmove to (12, 3)
4. Draw "Press q to quit" character by character

**6. _render_frame() -> void**

Orchestrate a complete frame render:
1. Call _draw_board()
2. If _game_over == 0: call _draw_piece()
3. Call _draw_score_panel()
4. If _game_over != 0: call _draw_game_over()
5. Load stdscr, call _wrefresh(stdscr)

**File structure:**
- Header comment block
- .section __TEXT,__text,regular,pure_instructions
- All 6 functions with .globl and .p2align 2
- .section __TEXT,__cstring for string literals ("Score", "Level", "Lines", "GAME OVER", "Press q to quit")
- .subsections_via_symbols at end

**Anti-patterns to avoid:**
- Do NOT use printw or mvwprintw -- they are variadic. Use wmove+waddch for everything.
- Do NOT forget to pass 3 arguments to wattr_on/wattr_off (win, attr, NULL). The third arg (opts) is NULL.
- Do NOT call wrefresh inside draw_board/draw_piece/draw_score_panel -- only call it once in render_frame after all drawing is done. Multiple wrefresh calls cause flicker.
- Do NOT use newwin. Use stdscr with computed coordinates (research recommendation for Phase 2).

<task type="auto">
  <name>Task 2: Create input.s with non-blocking input setup and key dispatch</name>
  <files>asm/input.s</files>
  <action>
Create asm/input.s with input initialization and key handling functions.

**1. _init_input() -> void**

Configure ncurses for non-blocking game input:
1. Load stdscr via GOT-indirect access (same pattern as Phase 1)
2. Call `bl _keypad` with (stdscr, 1) -- enable arrow key decoding. Without this, arrow keys produce escape sequences instead of KEY_LEFT/etc.
3. Call `bl _wtimeout` with (stdscr, 16) -- 16ms non-blocking timeout for ~60fps polling
4. Call `bl _noecho` -- suppress key echo (already called in main.s Phase 1, but safe to call again)
5. Call `bl _cbreak` -- character-at-a-time input (already called in main.s Phase 1, safe to re-call)

**2. _poll_input() -> w0 = key code (or -1 if no input)**

Simple wrapper around wgetch:
1. Load stdscr via GOT
2. Call `bl _wgetch` with stdscr as argument
3. Return result in w0 (key code or ERR=-1)

**3. _handle_input(w0=key_code) -> void**

Dispatch key press to the appropriate game action:

```
Key mapping:
  KEY_LEFT  (260) -> _try_move(-1, 0)   // move left
  KEY_RIGHT (261) -> _try_move(+1, 0)   // move right
  KEY_DOWN  (258) -> _soft_drop()        // soft drop (down one row)
  KEY_UP    (259) -> _try_rotate(1)      // rotate CW
  Space     (32)  -> _hard_drop()        // hard drop (instant)
  'z'       (122) -> _try_rotate(3)      // rotate CCW (3 = -1 mod 4)
  'q'       (113) -> set _game_over = 1  // quit game
```

Implementation:
1. Save the key code in a callee-saved register
2. Use a series of `cmp` + `b.eq` comparisons to branch to the appropriate handler
3. For each match, set up the function arguments and `bl` to the target function
4. If no match (unknown key): just return (ignore)

For _try_move calls, use `mov w0, #-1` (or `mov w0, #1`) and `mov w1, #0` (or `mov w1, #1`) before the bl.

Note about -1 in mov: ARM64 `mov w0, #-1` is actually encoded as `movn w0, #0` or can use `mov w0, #0xFFFFFFFF`. The assembler handles this. Alternatively, use: `mov w0, wzr / sub w0, w0, #1` if the assembler complains. But Apple's `as` should handle `mov w0, #-1` fine (it encodes as movn).

Actually, for signed -1 in w0, the cleanest approach is:
```asm
mov     w0, #-1         // Apple as handles this (encodes as ORN or MOVN)
```

**File structure:**
- Header comment block
- .section __TEXT,__text,regular,pure_instructions
- All 3 functions with .globl and .p2align 2
- .subsections_via_symbols at end
