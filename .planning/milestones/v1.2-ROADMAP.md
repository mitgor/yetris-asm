## Phase Details

### Phase 12: Code Cleanup
**Goal**: Codebase contains only live, referenced code with no dead functions, unused symbols, or stale data
**Depends on**: Phase 11 (v1.1 complete)
**Requirements**: CLEAN-01, CLEAN-02, CLEAN-03, CLEAN-04
**Success Criteria** (what must be TRUE):
  1. `_clear_lines` function is removed from board.s and the game still builds and runs with no regression (all line clears work via `_mark_lines` pipeline)
  2. Running `nm` on the binary shows no exported symbols that are never referenced by other files
  3. All string literals and data tables in data.s are referenced by at least one code path (no orphaned data)
  4. No redundant instruction sequences remain (e.g., back-to-back save/restore of same register, unnecessary moves)
**Plans**: 2 plans
- [x] 12-01-PLAN.md — Remove dead _clear_lines function, orphaned data strings, and audit .globl symbols
- [ ] 12-02-PLAN.md — Update version string to v1.2 and fix stale comments across codebase

### Phase 13: Binary Size Analysis
**Goal**: Binary size is measured per-section and per-file, easy optimizations are applied, and the C++ comparison is updated
**Depends on**: Phase 12 (cleanup changes binary size; measure after cleanup)
**Requirements**: SIZE-01, SIZE-02, SIZE-03, SIZE-04
**Success Criteria** (what must be TRUE):
  1. A documented breakdown shows bytes consumed by each Mach-O section (__TEXT, __DATA, __LINKEDIT) in the assembly binary
  2. Per-file contribution to total binary size is measured (which .s file contributes how many bytes)
  3. At least one concrete size optimization is implemented (literal pool sharing, data dedup, or alignment reduction) with before/after measurements
  4. The C++ vs assembly binary size comparison table is updated with post-cleanup, post-optimization numbers
**Plans**: TBD

### Phase 14: Research Documentation
**Goal**: The research writeup covers all v1.1 techniques and binary analysis findings, with a standalone deep-dive document
**Depends on**: Phase 13 (DOCS-03 needs binary analysis data)
**Requirements**: DOCS-01, DOCS-02, DOCS-03
**Success Criteria** (what must be TRUE):
  1. optimization-writeup.md contains new sections summarizing v1.1 techniques: subwindow composition, animation double-buffering, scoring pipeline, and hi-score file I/O
  2. A standalone research/v1.1-techniques.md exists with deep analysis of at least 4 v1.1 implementation techniques (not just summaries — code examples and design rationale)
  3. Binary size analysis findings (section breakdown, per-file contributions, optimization results) are documented in the research writeup
**Plans**: 2 plans
- [ ] 14-01-PLAN.md — Expand optimization-writeup.md with v1.1 technique summaries and binary size analysis findings
- [ ] 14-02-PLAN.md — Create standalone research/v1.1-techniques.md with deep analysis of 4 implementation techniques
