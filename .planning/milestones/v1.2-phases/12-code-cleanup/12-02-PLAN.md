<task type="auto">
  <name>Task 2: Scan for and remove redundant instruction sequences</name>
  <files>asm/board.s, asm/render.s, asm/main.s, asm/piece.s, asm/input.s, asm/menu.s, asm/animation.s, asm/layout.s, asm/hiscore.s, asm/timer.s, asm/random.s</files>
  <action>
1. **Scan all asm/*.s files for redundant instruction patterns.** Specifically search for:

   a. **Back-to-back stp/ldp of the same register set** — An `stp` that saves registers followed shortly by an `ldp` that restores the same registers with no intervening use. Example: `stp x19, x20, [sp, #-16]!` followed by `ldp x19, x20, [sp], #16` with no code between them that uses x19/x20.
      ```bash
      # Look for stp immediately followed by ldp of same registers (within 1-2 lines)
      grep -n "stp\|ldp" asm/*.s
      ```
      Review output for adjacent save/restore pairs with no intervening logic.

   b. **Redundant mov sequences** — A `mov` to a register followed by another `mov` from that register, where the intermediate register is unnecessary (e.g., `mov x1, x0` then `mov x2, x1` instead of `mov x2, x0`). Also `mov xN, xN` (no-op moves).
      ```bash
      grep -n "mov " asm/*.s
      ```
      Review output for unnecessary intermediary moves or self-moves.

   c. **Unnecessary push/pop patterns** — Saving callee-saved registers in a function prologue that never modifies them.
      Review each function's `stp` prologue: verify each saved register is actually written within the function body. If a register is saved but never written, remove it from the stp/ldp pair.

   d. **Redundant adrp+add sequences** — Loading the same address multiple times in a short span when the register still holds the value.
      ```bash
      grep -n "adrp\|add.*@PAGEOFF" asm/*.s
      ```
      Look for repeated adrp+add pairs for the same symbol within a basic block (no branch between them).

2. **For each redundancy found:**
   - Remove or consolidate the redundant instructions
   - Add a brief comment explaining the consolidation if the change is non-obvious
   - Ensure correctness: verify the register is truly unused between the save/restore or that the intermediate mov is genuinely unnecessary

3. **If no redundancies are found:**
   - This is a valid outcome (the codebase may already be clean)
   - Document in the done list that a thorough scan was performed and no redundancies were found

4. Build: run `make asm` to confirm the binary still builds and links correctly after any changes.

5. Smoke test: run `make asm-run`, start a game, play through several line clears, verify no regression.
  </action>
