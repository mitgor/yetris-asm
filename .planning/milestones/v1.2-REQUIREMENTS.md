# Requirements: yetris-asm v1.2

**Defined:** 2026-02-27
**Core Value:** A fully playable Tetris clone in ARM assembly that is measurably smaller and faster than the C++ original, with a documented writeup explaining every technique and tradeoff.

## v1.2 Requirements

Requirements for polish and cleanup milestone. Each maps to roadmap phases.

### Code Cleanup

- [x] **CLEAN-01**: Dead code _clear_lines removed from board.s with no regression
- [x] **CLEAN-02**: All exported symbols audited — unused .globl symbols removed or made local
- [x] **CLEAN-03**: Unused data strings/tables identified and removed from data.s
- [x] **CLEAN-04**: Small code quality improvements applied (redundant instructions, alignment, clarity)

### Binary Analysis

- [x] **SIZE-01**: Binary size breakdown by Mach-O section (__TEXT, __DATA, __LINKEDIT) documented
- [x] **SIZE-02**: Per-file contribution to binary size measured and documented
- [x] **SIZE-03**: Easy size optimizations identified and implemented (literal pool sharing, data dedup, alignment)
- [x] **SIZE-04**: Before/after binary size comparison with C++ baseline updated

### Research Documentation

- [x] **DOCS-01**: optimization-writeup.md expanded with v1.1 technique summaries (subwindows, animation, scoring)
- [x] **DOCS-02**: Standalone research/v1.1-techniques.md created with deep analysis of animation double-buffering, scoring pipeline, subwindow composition, Darwin syscall file I/O
- [x] **DOCS-03**: Binary size analysis findings documented in research writeup

## Out of Scope

| Feature | Reason |
|---------|--------|
| New gameplay features | v1.2 is polish only — no new game mechanics |
| Player profiles | Massive complexity; deferred |
| Configurable keybindings | Requires file parsing; compile-time keys sufficient |
| Cross-platform | macOS only (Darwin syscall conventions) |
| Rewriting ncurses | Use as-is for terminal abstraction |
| Performance benchmarking | v1.0 already measured frame timing; v1.2 focuses on size not speed |
