# Requirements: yetris-asm v1.1

**Defined:** 2026-02-27
**Core Value:** A fully playable Tetris clone in ARM assembly that is measurably smaller and faster than the C++ original, with a documented writeup explaining every technique and tradeoff.

## v1.1 Requirements

Requirements for visual polish and gameplay milestone. Each maps to roadmap phases.

### Layout & Subwindows

- [x] **LAYOUT-01**: Game screen uses ncurses subwindows (newwin/derwin) instead of direct stdscr drawing
- [x] **LAYOUT-02**: Game layout matches C++ original's 80x24 grid with exact panel column positions (leftmost=12w, board=22w, middle_right=10w, rightmost=fills)
- [x] **LAYOUT-03**: Hold panel displays in leftmost window (4 rows high, titled "Hold")
- [x] **LAYOUT-04**: Score panel displays Hi-Score, Score, and Level in leftmost window below Hold
- [x] **LAYOUT-05**: Next piece panel displays in middle-right window (titled "Next")
- [x] **LAYOUT-06**: Statistics panel displays in rightmost window with piece counts, line clear counts, timer, and version string (titled "Statistics")
- [x] **LAYOUT-07**: Menu screen uses ncurses subwindows with logo window and menu window matching C++ layout

### Visual Polish

- [x] **VISUAL-01**: ASCII art multi-line "YETRIS" logo renders on main menu, horizontally centered
- [x] **VISUAL-02**: All window borders use ACS box-drawing characters (ACS_VLINE, ACS_HLINE, ACS_ULCORNER, etc.) with color-coded shadow effect (left/top brighter, right/bottom dimmer)
- [x] **VISUAL-03**: Window titles ("Hold", "Next", "Statistics", "Paused") render in highlight color over border
- [x] **VISUAL-04**: UI labels (Score, Level, Lines, Hi-Score, stats) render in highlight color (cyan)
- [x] **VISUAL-05**: Menu items use highlight color for selected item, with colored first-letter mnemonics
- [x] **VISUAL-06**: Additional color pairs initialized beyond piece colors (highlight, dim, dim-dim, textbox)
- [x] **VISUAL-07**: Pause overlay renders as bordered subwindow with menu items (Resume, Quit to Menu, Quit Game)
- [x] **VISUAL-08**: Game over overlay renders with styled text

### Background Animations

- [x] **ANIM-01**: Fire animation — particle system with heat propagation, cooling map, ASCII grayscale, red/yellow/white colors, 100ms update rate
- [x] **ANIM-02**: Water animation — double-buffer wave propagation, ripple simulation, blue/cyan/white colors, 300ms update rate
- [x] **ANIM-03**: Snakes animation — falling green "snake" entities (head='@', body='o'), Matrix-style, 50ms update rate, max 50 snakes
- [x] **ANIM-04**: Game of Life animation — Conway's B3/S23 rules, yellow living cells, 200ms update rate
- [x] **ANIM-05**: Random animation selection at startup for both menu background and game background
- [x] **ANIM-06**: Animations run behind menu screen (below logo, behind menu window)
- [x] **ANIM-07**: Animations run behind game board during gameplay

### Modern Scoring

- [x] **SCORE-01**: Line clear scores multiplied by level (Single=100*lvl, Double=300*lvl, Triple=500*lvl, Tetris=800*lvl)
- [x] **SCORE-02**: Combo system — 50 * combo_count * level added for consecutive line-clearing locks, counter resets on non-clearing lock
- [x] **SCORE-03**: Back-to-back bonus — 1.5x multiplier for consecutive difficult clears (Tetris or T-spin)
- [x] **SCORE-04**: T-spin detection using 3-corner rule (after T rotation, 3+ of 4 diagonal corners occupied)
- [x] **SCORE-05**: T-spin scoring values (T-spin zero=400*lvl, Single=800*lvl, Double=1200*lvl, Triple=1600*lvl)
- [x] **SCORE-06**: Perfect clear detection (board completely empty after line clear) with bonus scoring
- [x] **SCORE-07**: Soft drop scoring — 1 point per cell dropped
- [x] **SCORE-08**: Hard drop scoring — 2 points per cell dropped

### Line Clear Animation

- [x] **CLEAR-01**: Full rows flash with special marker characters (e.g., '::') before removal
- [x] **CLEAR-02**: 200ms visual delay between flash and row removal, matching C++ behavior

### Hi-Score Persistence

- [x] **HISCORE-01**: Top score saved to ~/.yetris-hiscore as 4-byte uint32 on game over (if new high)
- [x] **HISCORE-02**: Top score loaded from file on startup (defaults to 0 if missing/unreadable)
- [x] **HISCORE-03**: Hi-Score displayed in score panel with "Hi-Score" label and "(none)" when zero

## Out of Scope

| Feature | Reason |
|---------|--------|
| Player profiles | Massive complexity (INI parsing, directory management); single global hi-score sufficient |
| Configurable keybindings from file | Requires file parsing and string matching; compile-time keys sufficient |
| Theme system | C++ theme system spans ~200 lines; hardcoded defaults match C++ |
| Top-10 leaderboard | Single hi-score sufficient for v1.1 scope |
| All-spin detection (S/Z/L/J spins) | Only T-spin is standard modern Tetris |
| wnoutrefresh batching | Start with wrefresh; upgrade only if flickering occurs |
