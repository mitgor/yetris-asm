**Part A: Create asm/layout.s** -- a new assembly file with 4 exported functions for window lifecycle management.

File structure:
1. Standard file header comment (match style of other asm/*.s files)
2. .section __TEXT,__text
3. Four functions: _init_game_layout, _destroy_game_layout, _init_menu_layout, _destroy_menu_layout

**_init_game_layout:**
- Prologue: save x19-x24 and x29/x30 (need 6 callee-saved registers to hold parent WINDOW* pointers across derwin calls)
- Create main: bl _newwin with args (w0=24, w1=80, w2=0, w3=0), store result to _win_main via adrp+str PAGE/PAGEOFF, keep in x19
- Create leftmost: bl _derwin(x19, 24, 12, 0, 0), store to _win_leftmost, keep in x20
- Create hold: bl _derwin(x20, 4, 12, 0, 0), store to _win_hold
- Create score: bl _derwin(x20, 20, 12, 4, 0), store to _win_score
- Create middle_left: bl _derwin(x19, 22, 22, 0, 12), store to _win_middle_left, keep in x21
- Create board: bl _derwin(x21, 22, 22, 0, 0), store to _win_board
- Create middle_right: bl _derwin(x19, 4, 10, 0, 34), store to _win_middle_right
- Create rightmost: bl _derwin(x19, 24, 35, 0, 44), store to _win_rightmost
- Create pause: bl _derwin(x19, 6, 40, 11, 20), store to _win_pause
- Epilogue: restore registers, ret

**_destroy_game_layout:**
- Prologue: save x19, x29/x30
- Delete children first (reverse creation order): pause, rightmost, middle_right, board, middle_left, score, hold, leftmost, main
- For each: load pointer via adrp+ldr, cbz to skip if NULL, bl _delwin
- Zero all pointers after deletion: str xzr to each _win_* slot
- Epilogue: restore, ret

**_init_menu_layout:**
- Create menu_main: bl _newwin(24, 80, 0, 0), store to _win_menu_main, keep in x19
- Create menu_logo: bl _derwin(x19, 9, 80, 0, 0), store to _win_menu_logo
- Create menu_items: bl _derwin(x19, 13, 28, 10, 24), store to _win_menu_items
  (Note: 80/3=26 in integer division; menu_x=26-2=24, menu_w=26+2=28, menu_y=9+1=10, menu_h=24-9-2=13)

**_destroy_menu_layout:**
- Delete in reverse: menu_items, menu_logo, menu_main
- Zero all pointers

CRITICAL patterns to follow:
- All local data access uses adrp+add @PAGE/@PAGEOFF (same binary, NOT GOT) -- _win_main etc. are local symbols
- All function calls use `bl` prefix (Darwin convention: all C symbols have underscore prefix: _newwin, _derwin, _delwin)
- derwin argument order: x0=parent WINDOW*, w1=nlines, w2=ncols, w3=begin_y, w4=begin_x
- newwin argument order: w0=nlines, w1=ncols, w2=begin_y, w3=begin_x
- L-prefix for all internal branch targets (e.g., Lskip_pause_del)
- .globl and .p2align 2 before each exported function
- End with .subsections_via_symbols
