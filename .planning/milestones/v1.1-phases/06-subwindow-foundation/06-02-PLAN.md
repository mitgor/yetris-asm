**Part J: Rewrite _render_frame for subwindow refresh protocol**

Replace the current single-wrefresh(stdscr) pattern with the ncurses batch refresh protocol:

```
_render_frame:
    // 1. Clear + wnoutrefresh all container windows (parent before child)
    //    Load each container, werase, wnoutrefresh
    //    Containers: _win_main, _win_leftmost, _win_middle_left, _win_middle_right, _win_rightmost

    // 2. Draw content into leaf windows (each function handles its own wnoutrefresh)
    bl _draw_board          // draws into _win_board, calls wnoutrefresh

    // 3. Check game_over / paused (same logic as current)
    //    If running: draw ghost + piece into _win_board
    //    If paused: draw pause overlay into _win_pause

    // 4. Draw panels (each handles its own wnoutrefresh)
    bl _draw_score_panel
    bl _draw_next_panel
    bl _draw_hold_panel
    bl _draw_stats_panel

    // 5. Board wnoutrefresh (after ghost+piece drawn on top)
    //    Note: _draw_board already called wnoutrefresh, but ghost/piece drew after it,
    //    so call wnoutrefresh on _win_board again here
    ldr x0, [_win_board ptr]
    bl _wnoutrefresh

    // 6. If game_over: draw game_over overlay (on _win_board, wnoutrefresh again)

    // 7. Single terminal flush
    bl _doupdate
```

Remove the old wrefresh(stdscr) call. Remove the werase(stdscr) from _draw_board (containers are erased in _render_frame). Each leaf draw function should call wnoutrefresh at its end.
