**Step 2: Expand pause gate in asm/input.s _handle_input** to support pause menu navigation:

Currently (lines 122-133):
```
// Pause gate: if paused, only allow p/q/ESC
adrp    x8, _is_paused@PAGE
ldrb    w8, [x8, _is_paused@PAGEOFF]
cbz     w8, Lcheck_left             // not paused, proceed normally

```

Replace with expanded gate that also handles UP, DOWN, ENTER for pause menu:

```asm
// Pause gate: if paused, handle pause menu navigation
adrp    x8, _is_paused@PAGE
ldrb    w8, [x8, _is_paused@PAGEOFF]
cbz     w8, Lcheck_left             // not paused, proceed normally

// Allow 'p' to unpause
cmp     w19, #112                   // 'p'
b.eq    Lpause_resume               // unpause (same as selecting Resume)

// KEY_UP (259): decrement _pause_selection (clamp to 0)
mov     w8, #259
cmp     w19, w8
b.ne    Lpause_check_down
adrp    x8, _pause_selection@PAGE
add     x8, x8, _pause_selection@PAGEOFF
ldrb    w9, [x8]
cbz     w9, Lhandle_done            // already at 0
sub     w9, w9, #1
strb    w9, [x8]
b       Lhandle_done

Lpause_check_down:
// KEY_DOWN (258): increment _pause_selection (clamp to 2)
mov     w8, #258
cmp     w19, w8
b.ne    Lpause_check_enter
adrp    x8, _pause_selection@PAGE
add     x8, x8, _pause_selection@PAGEOFF
ldrb    w9, [x8]
cmp     w9, #2
b.ge    Lhandle_done                // already at max
add     w9, w9, #1
strb    w9, [x8]
b       Lhandle_done

Lpause_check_enter:
// ENTER (10) or KEY_ENTER (343): activate selected pause menu item
cmp     w19, #10
b.eq    Lpause_activate
mov     w8, #343
cmp     w19, w8
b.eq    Lpause_activate

// 'q' or ESC: quit game
cmp     w19, #113                   // 'q'
b.eq    Lset_quit
cmp     w19, #27                    // ESC
b.eq    Lset_quit
b       Lhandle_done                // block all other keys

Lpause_activate:
// Dispatch based on _pause_selection
adrp    x8, _pause_selection@PAGE
ldrb    w9, [x8, _pause_selection@PAGEOFF]
cmp     w9, #0
b.eq    Lpause_resume               // Resume
cmp     w9, #1
b.eq    Lpause_quit_to_menu         // Quit to Main Menu
// w9 == 2: Quit Game
b       Lset_quit

Lpause_resume:
// Unpause: set _is_paused = 0, reset gravity timer, reset _pause_selection
adrp    x8, _is_paused@PAGE
strb    wzr, [x8, _is_paused@PAGEOFF]
adrp    x8, _pause_selection@PAGE
strb    wzr, [x8, _pause_selection@PAGEOFF]
bl      _get_time_ms
adrp    x8, _last_drop_time@PAGE
str     x0, [x8, _last_drop_time@PAGEOFF]
b       Lhandle_done

Lpause_quit_to_menu:
// Set _game_over = 1 to exit game loop, which will return to menu via main.s
// The main.s Lgame_over_screen -> Lreturn_to_menu handles the transition
adrp    x8, _game_over@PAGE
mov     w9, #1
strb    w9, [x8, _game_over@PAGEOFF]
// Reset pause state
adrp    x8, _is_paused@PAGE
strb    wzr, [x8, _is_paused@PAGEOFF]
adrp    x8, _pause_selection@PAGE
strb    wzr, [x8, _pause_selection@PAGEOFF]
b       Lhandle_done
```

IMPORTANT: The existing Lcheck_p code (lines 208-224) that toggles _is_paused via `eor w9, w9, #1` is now partially redundant. When 'p' is pressed during pause, it should go to Lpause_resume (which handles unpause cleanly). When 'p' is pressed during gameplay (not paused), the existing Lcheck_p code handles the pause-on correctly. So:
- Keep the Lcheck_p label and its code for the "press p to pause" case (when not paused)
- The pause gate now handles 'p' during pause by jumping to Lpause_resume instead of Lcheck_p

_draw_game_over:
    stp     x20, x19, [sp, #-16]!
    stp     x29, x30, [sp, #-16]!

**Step 2: Add colored first-letter mnemonics in asm/menu.s _menu_frame:**

Currently, menu items are drawn as full strings with A_REVERSE when selected. The C++ original renders the first character of each menu item in hilite_text (bold cyan), and the rest in normal text color. When selected, the entire item uses A_REVERSE.

Replace the menu item drawing loop (the section around Lmenu_item_loop that draws 3 action items). The current code:
1. Positions cursor at (row, 1)
2. Optionally applies A_REVERSE for selected item
3. Draws the full string via waddstr
4. Optionally removes A_REVERSE

New logic for each item:
1. Position cursor at (row, 1)
2. If selected: apply A_REVERSE for the whole item, draw full string, remove A_REVERSE
3. If NOT selected: apply hilite_hilite_text (bold cyan), draw first char via waddch, remove hilite_hilite_text, then draw rest of string via waddstr (which means we need a pointer past the first character, i.e., string_ptr+1)

For the "not selected" case, the string pointer from _menu_items[i] gives us the full string. First char = load byte at [ptr], rest = ptr+1.

```asm
// For non-selected items (when w21 != w20):
// Draw first letter in bold cyan
mov     x0, x19
mov     w1, #0x0A00             // COLOR_PAIR(10)
movk    w1, #0x0020, lsl #16   // | A_BOLD
mov     x2, #0
bl      _wattr_on

// Load string pointer
uxtw    x8, w21
ldr     x1, [x22, x8, lsl #3]  // full string ptr
ldrb    w1, [x1]               // first character
mov     x0, x19
bl      _waddch

// Turn off bold cyan
mov     x0, x19
mov     w1, #0x0A00
movk    w1, #0x0020, lsl #16
mov     x2, #0
bl      _wattr_off

// Draw rest of string (ptr + 1)
uxtw    x8, w21
ldr     x1, [x22, x8, lsl #3]
add     x1, x1, #1             // skip first char
mov     x0, x19
bl      _waddstr
```

For the selected case (A_REVERSE), keep the existing behavior -- draw the full string with A_REVERSE. No colored first letter needed when selected since A_REVERSE covers the whole string.

Apply this pattern only to the 3 action items (Start Game, Help, Quit -- indices 0-2). The 5 settings items (indices 3-7) remain as-is since they use label+value format, not single action strings.
