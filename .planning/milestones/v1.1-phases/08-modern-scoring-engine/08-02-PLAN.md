**D. Create `_user_soft_drop` in `asm/input.s` and wire KEY_DOWN:**

```asm
// ============================================================================
// _user_soft_drop: User-initiated soft drop with scoring
// void _user_soft_drop(void) -> w0=1 moved, w0=0 locked
//
// Wrapper around _soft_drop that awards 1 point per cell dropped.
// Only called from KEY_DOWN in input handler. Gravity calls _soft_drop directly.
// ============================================================================
.globl _user_soft_drop
.p2align 2
_user_soft_drop:
    stp     x29, x30, [sp, #-16]!
    mov     x29, sp

    // Try to move down
    mov     w0, #0                  // dx = 0
    mov     w1, #1                  // dy = +1
    bl      _try_move

    cbz     w0, Lusd_blocked        // blocked -> lock and spawn

    // Moved successfully: award 1 point
    adrp    x8, _score@PAGE
    add     x8, x8, _score@PAGEOFF
    ldr     w9, [x8]
    add     w9, w9, #1
    str     w9, [x8]

    mov     w0, #1                  // return 1 (moved)
    b       Lusd_epilogue

Lusd_blocked:
    // Blocked: lock and spawn (same as _soft_drop)
    bl      _lock_piece
    bl      _spawn_piece

    mov     w0, #0                  // return 0 (locked)

Lusd_epilogue:
    ldp     x29, x30, [sp], #16
    ret
```


```asm
    bl      _user_soft_drop
```

This ensures gravity (which calls `_soft_drop` in main.s) does NOT award soft drop points.

**A. T-spin detection -- add AFTER writing piece to board, BEFORE `bl _clear_lines`:**

The detection must happen after the piece is written to the board (so we can check the board state) but before lines are cleared (so the corners are still intact).

Actually, the corners are diagonal to the pivot which is a T-piece cell (piece_type+1), not a line clear cell. The corners are NOT part of the cleared rows. So detection can happen after `bl _clear_lines` too. But for clarity and correctness, do it BEFORE `bl _clear_lines`:

Insert T-spin detection between the `_can_hold` reset and the `bl _clear_lines` call:

```asm
    // T-spin detection: check if piece_type==6 (T) AND _last_was_rotation==1
    adrp    x8, _is_tspin@PAGE
    strb    wzr, [x8, _is_tspin@PAGEOFF]       // default: not a T-spin

    cmp     w19, #6                             // w19 = piece_type (loaded earlier)
    b.ne    Ltspin_done                         // not T-piece -> skip

    adrp    x8, _last_was_rotation@PAGE
    ldrb    w8, [x8, _last_was_rotation@PAGEOFF]
    cbz     w8, Ltspin_done                     // last action was not rotation -> skip

    // T-piece placed after rotation -- check 3-corner rule
    // Pivot is at grid position (2,2) in piece_data
    // Board pivot coords: pivot_x = piece_x + 2, pivot_y = piece_y + 2
    add     w8, w21, #2                         // w8 = pivot_x (w21 = piece_x)
    add     w9, w22, #2                         // w9 = pivot_y (w22 = piece_y)

    // Load board base for corner checks
    adrp    x10, _board@PAGE
    add     x10, x10, _board@PAGEOFF

    mov     w11, #0                             // occupied_count = 0

    // Corner 1: (pivot_y-1, pivot_x-1)
    sub     w12, w9, #1                         // cy
    sub     w13, w8, #1                         // cx
    // Bounds check: out-of-bounds counts as occupied
    cmp     w13, #0
    b.lt    Lc1_occ
    cmp     w13, #10
    b.ge    Lc1_occ
    cmp     w12, #0
    b.lt    Lc1_occ
    cmp     w12, #20
    b.ge    Lc1_occ
    // In bounds: check board cell
    mov     w14, #10
    mul     w14, w12, w14
    add     w14, w14, w13
    uxtw    x14, w14
    ldrb    w15, [x10, x14]
    cbz     w15, Lc1_done                       // empty -> not occupied
Lc1_occ:
    add     w11, w11, #1
Lc1_done:

    cmp     w13, #10
    b.ge    Lc2_occ
    cmp     w12, #0
    b.lt    Lc2_occ
    cmp     w12, #20
    b.ge    Lc2_occ
    mov     w14, #10
    mul     w14, w12, w14
    add     w14, w14, w13
    uxtw    x14, w14
    ldrb    w15, [x10, x14]
    cbz     w15, Lc2_done
Lc2_occ:
    add     w11, w11, #1
Lc2_done:

    sub     w13, w8, #1

    cmp     w13, #10
    b.ge    Lc3_occ
    cmp     w12, #0

Lc3_occ:

Lc3_done:
```

**B. Modify scoring pipeline in `_lock_piece` -- integrate T-spin scoring:**

In the scoring pipeline (added by Plan 08-01), after `bl _clear_lines` returns w0 = lines_cleared (saved in w24):

1. **T-spin with 0 lines:** Change the "if lines == 0, reset combo, skip scoring" logic. Before resetting combo, check if `_is_tspin == 1`. If T-spin with 0 lines, award `_tspin_score_table[0] * level = 400 * level` to `_score`, then proceed to combo reset (T-spin zero does NOT contribute to combo or B2B). Pseudocode:
   ```
   if lines == 0:
       if _is_tspin:
           _score += _tspin_score_table[0] * _level  // 400 * level
       _combo_count = 0
       goto invisible_check
   ```

2. **Base score with T-spin override:** If `_is_tspin == 1` AND `lines > 0`, use `_tspin_score_table[lines]` instead of `_score_table[lines-1]`. Note the indexing: T-spin table index 0 = zero lines, 1 = single, 2 = double, 3 = triple. So for T-spin with lines > 0, index = lines (not lines-1). For normal clears, index = lines-1 as before:
   ```
   if _is_tspin && lines > 0:
       base = _tspin_score_table[lines] * _level    // index = lines (1,2,3)
   else:
       base = _score_table[lines-1] * _level         // index = lines-1 (0,1,2,3)
   ```

3. **B2B difficult check:** Update the `is_difficult` logic. Currently (from Plan 08-01): `is_difficult = (lines == 4)`. Change to: `is_difficult = (lines == 4) || (_is_tspin == 1 && lines > 0)`. In assembly:
   ```asm
   // w12 = is_difficult
   mov     w12, #0
   cmp     w24, #4                     // Tetris?
   b.eq    Lset_difficult
   adrp    x8, _is_tspin@PAGE
   ldrb    w8, [x8, _is_tspin@PAGEOFF]
   cbz     w8, Lnot_difficult          // not T-spin -> not difficult
   // T-spin with lines > 0 is difficult (we already know lines > 0 at this point)
   Lset_difficult:
   mov     w12, #1
   Lnot_difficult:
   ```

**IMPORTANT:** Do NOT use `bl` (branch and link) for the corner checks. All the corner check code should use inline branches (`b.xx`) with local labels, since this runs in the middle of `_lock_piece` which already manages its own callee-saved registers. The research suggested `bl Lcheck_corner` but that would clobber x30 (link register). Use fully inlined checks as shown above.

**ALSO:** Be careful about register allocation. The scoring pipeline uses w8-w15 as scratch. The T-spin detection (before `bl _clear_lines`) also uses w8-w15. Since `bl _clear_lines` clobbers all caller-saved registers, the T-spin detection results must be stored to memory (`_is_tspin`) before calling `_clear_lines`, and reloaded after. This is already handled by writing `_is_tspin` to memory.
