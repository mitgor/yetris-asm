  <action>
Make three changes to `asm/board.s`:

**A. Simplify `_clear_lines` -- remove score computation:**

In `_clear_lines`, at label `Lclear_done`, remove the score addition block. Specifically, delete the lines that:
- Load `_score_table` and add `_score_table[lines-1]` to `_score`

The function should still:
1. Scan/clear full rows (existing NEON detection + shift logic -- KEEP THIS UNCHANGED)
2. Update `_lines_cleared` total -- KEEP
3. Recompute `_level` from thresholds -- KEEP
4. Update line-type stats (`_stats_singles`/`_stats_doubles`/`_stats_triples`/`_stats_tetris`) -- KEEP
5. Return `lines_cleared` count in w0 -- KEEP

Remove ONLY the 6-line block at `Lclear_done` that loads `_score_table`, indexes by `(lines-1)`, and adds to `_score`. The block to remove is:
```asm
    adrp    x8, _score_table@PAGE
    add     x8, x8, _score_table@PAGEOFF
    sub     w9, w21, #1
    ldr     w10, [x8, w9, uxtw #2]
    ... (load _score, add, store)
```

**B. Restructure `_lock_piece` -- centralize all scoring:**

The current `_lock_piece` does: write piece to board -> add +10 -> stats -> can_hold -> clear_lines -> invisible mode.

Change it to: write piece to board -> stats -> can_hold -> clear_lines -> SCORING ENGINE -> invisible mode.

After `bl _clear_lines` returns with `w0 = lines_cleared`:

1. **Remove the flat +10 lock bonus.** Delete the block at line ~238-243 that adds 10 to `_score`. Modern guideline does not have a lock bonus.

2. **Save lines_cleared** in a callee-saved register (use w24 as currently done, or w25 if needed).

3. **If lines_cleared == 0:** Reset `_combo_count` to 0 and skip all scoring. Jump to invisible mode check.

4. **If lines_cleared > 0:** Execute the scoring pipeline:

   a. **Base score = `_score_table[lines-1] * _level`:**
      ```asm
      adrp    x8, _score_table@PAGE
      add     x8, x8, _score_table@PAGEOFF
      sub     w9, w24, #1              // index = lines - 1
      ldr     w10, [x8, w9, uxtw #2]  // w10 = base (100/300/500/800)
      adrp    x8, _level@PAGE
      ldr     w11, [x8, _level@PAGEOFF]
      mul     w10, w10, w11            // w10 = base * level
      ```
      w10 holds the line_clear_score.

   b. **Back-to-back check:**
      Determine if current clear is "difficult." For now (Plan 01 without T-spin), difficult = (lines_cleared == 4, i.e., Tetris). Plan 08-02 will add T-spin as difficult.
      ```asm
      // w_is_difficult: 1 if Tetris (lines==4), 0 otherwise
      cmp     w24, #4
      cset    w12, eq                   // w12 = is_difficult (1 if Tetris)
      ```
      If `_b2b_active == 1` AND `w12 == 1` (current is difficult), add 50% bonus:
      ```asm
      add     w10, w10, w10, lsr #1    // score = score * 1.5
      ```
      Then update `_b2b_active`:
      - If `w12 == 1` (difficult): set `_b2b_active = 1`
      - If `w12 == 0` (not difficult) AND `w24 > 0` (lines were cleared): set `_b2b_active = 0` (chain broken)

   c. **Add line clear score to `_score`:**
      ```asm
      adrp    x8, _score@PAGE
      add     x8, x8, _score@PAGEOFF
      ldr     w11, [x8]
      add     w11, w11, w10
      str     w11, [x8]
      ```

   d. **Combo bonus:**
      Increment `_combo_count`, then add `50 * _combo_count * _level` to `_score`:
      ```asm
      adrp    x8, _combo_count@PAGE
      add     x8, x8, _combo_count@PAGEOFF
      ldr     w9, [x8]
      add     w9, w9, #1
      str     w9, [x8]                  // _combo_count++

      mov     w10, #50
      mul     w10, w9, w10              // combo_count * 50
      adrp    x11, _level@PAGE
      ldr     w11, [x11, _level@PAGEOFF]
      mul     w10, w10, w11             // combo_count * 50 * level

      adrp    x8, _score@PAGE
      add     x8, x8, _score@PAGEOFF
      ldr     w11, [x8]
      add     w11, w11, w10
      str     w11, [x8]
      ```

   e. **Perfect clear check:**
      After clearing lines, scan the entire 200-byte board to check if it's empty. Use a scalar loop (simpler, and perfect clears are extremely rare):
      ```asm
      adrp    x8, _board@PAGE
      add     x8, x8, _board@PAGEOFF
      mov     w9, #0                    // byte index
      mov     w13, #0                   // accumulator (OR all bytes)
      Lpc_scan:
          ldrb    w14, [x8, x9]
          orr     w13, w13, w14
          add     w9, w9, #1
          cmp     w9, #200
          b.lt    Lpc_scan
      cbnz    w13, Lpc_not_perfect     // any non-zero byte -> not perfect
      ```
      If perfect clear (w13 == 0): add `_perfect_clear_table[lines-1] * level` to `_score`:
      ```asm
      adrp    x8, _perfect_clear_table@PAGE
      add     x8, x8, _perfect_clear_table@PAGEOFF
      sub     w9, w24, #1
      ldr     w10, [x8, w9, uxtw #2]   // base perfect clear bonus
      adrp    x8, _level@PAGE
      ldr     w11, [x8, _level@PAGEOFF]
      mul     w10, w10, w11             // bonus * level
      adrp    x8, _score@PAGE
      add     x8, x8, _score@PAGEOFF
      ldr     w11, [x8]
      add     w11, w11, w10
      str     w11, [x8]
      ```

The full scoring flow in `_lock_piece` after `bl _clear_lines` should be:
```
lines = _clear_lines()
if lines == 0:
    _combo_count = 0
    goto invisible_check
// Base score
score = _score_table[lines-1] * _level
// B2B
is_difficult = (lines == 4)   // Plan 02 adds: || _is_tspin
if _b2b_active && is_difficult:
    score += score >> 1        // 1.5x
if is_difficult:
    _b2b_active = 1
else:
    _b2b_active = 0            // lines > 0 but not difficult breaks chain
// Add line clear score
_score += score
// Combo
_combo_count++
_score += 50 * _combo_count * _level
// Perfect clear
if board_is_empty():
    _score += _perfect_clear_table[lines-1] * _level
```

NOTE: The stack frame for `_lock_piece` is currently 64 bytes with 6 callee-saved registers (x19-x24). This should be sufficient since we reuse w24 for lines_cleared (already done) and use caller-saved registers (w8-w15) for intermediate computations. Do NOT expand the stack frame unless absolutely necessary.

```asm
    add     x8, x8, _b2b_active@PAGEOFF

```

**IMPORTANT ANTI-PATTERNS TO AVOID:**
- Do NOT keep any score addition in `_clear_lines`. All scoring belongs in `_lock_piece`.

- The combo counter increment happens BEFORE computing combo bonus (so first consecutive clear gets combo_count=1, bonus=50*1*level).
- A lock with 0 lines cleared does NOT break the B2B chain -- only a non-difficult clear with lines > 0 breaks it.
