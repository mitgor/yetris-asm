**asm/hiscore.s** -- Create new file with two public functions:

File header: Standard yetris-asm file header documenting the module (file I/O for hi-score persistence via Darwin ARM64 syscalls).

**`_load_hiscore`**: Called once at startup. Loads 4-byte uint32 from `~/.yetris-hiscore` into `_hiscore`. On any failure, leaves `_hiscore` at default 0.

Implementation:
1. Save callee-saved registers: stp x20, x19 then stp x29, x30. Set frame pointer.
2. Allocate 256 bytes on stack for path buffer: `sub sp, sp, #256`
3. Call `_getenv` with `_str_home_env` ("HOME"). If NULL (cbz x0), jump to bail.
4. Copy HOME string to stack buffer (x19 = sp) using byte-copy loop: `ldrb w8, [x0], #1` / `strb w8, [x1], #1` / `cbnz w8, loop`. Back up 1 byte over the NUL terminator.
5. Append `_str_hiscore_suffix` ("/.yetris-hiscore") using same byte-copy loop (including the NUL terminator).
6. Open file: `mov x0, x19` (path), `mov w1, #0` (O_RDONLY), `mov w2, #0`, `mov x16, #5` (syscall open), `svc #0x80`. Check carry flag: `b.cs Lload_bail`.
7. Save fd in w20. Allocate 16-byte read buffer on stack: `sub sp, sp, #16`, zero it: `str wzr, [sp]`.
8. Read 4 bytes: `mov w0, w20` (fd), `mov x1, sp` (buf), `mov x2, #4` (count), `mov x16, #3` (syscall read), `svc #0x80`. On carry-set error, jump to close.
9. Load the 4-byte value from stack buffer: `ldr w8, [sp]`. Store to `_hiscore`: `adrp x9, _hiscore@PAGE` / `str w8, [x9, _hiscore@PAGEOFF]`.
10. Free read buffer: `add sp, sp, #16`. Close fd: `mov w0, w20`, `mov x16, #6`, `svc #0x80`.
11. Bail label: free path buffer `add sp, sp, #256`, restore registers, `ret`.

Important: The close path and bail path must be separate. If open succeeded, always close before bailing. Structure:
- `Lload_close:` -- free read buffer, close fd, then fall through to bail
- `Lload_bail:` -- free path buffer, restore frame, ret

**`_save_hiscore`**: Called from game-over path. Writes `_hiscore` as 4-byte uint32 to `~/.yetris-hiscore`.

Implementation:
1. Same register save and 256-byte stack allocation as load.
2. Same HOME resolution and path construction (getenv + byte-copy HOME + append suffix).
3. Open for writing: `mov x0, x19` (path), `mov w1, #0x0601` (O_WRONLY=0x1 | O_CREAT=0x200 | O_TRUNC=0x400), `mov w2, #0x1A4` (0644 octal), `mov x16, #5`, `svc #0x80`. On carry-set, jump to bail.
4. Save fd in w20. Allocate 16-byte write buffer: `sub sp, sp, #16`. Load `_hiscore` value and store to stack buffer: `adrp x8, _hiscore@PAGE` / `ldr w8, [x8, _hiscore@PAGEOFF]` / `str w8, [sp]`.
5. Write 4 bytes: `mov w0, w20`, `mov x1, sp`, `mov x2, #4`, `mov x16, #4` (syscall write), `svc #0x80`.
6. Free write buffer. Close fd. Free path buffer. Restore registers. Ret.

**Important implementation notes:**
- All function labels must be `.globl` and `.p2align 2`.
- Stack must stay 16-byte aligned at all `bl` sites. 256 is 16-aligned. The stp pairs push 16+16=32 bytes.
- Use callee-saved registers x19 (path buffer base) and x20 (fd) to survive across `bl _getenv`.
- Darwin ARM64 syscall error convention: carry flag set = error, x0 = errno. Always check with `b.cs`.
- Do NOT use libc _open/_read/_write/_close -- use raw `svc #0x80` syscalls matching the existing codebase pattern in main.s.
- `mov w1, #0x0601` -- the assembler accepts this 16-bit immediate directly (fits in a single MOV instruction since bit pattern 0x0601 can be expressed).

Lgame_over_screen:
    // Check if current score beats hi-score, save if so (Phase 11)
    adrp    x8, _score@PAGE
    ldr     w9, [x8, _score@PAGEOFF]
    adrp    x8, _hiscore@PAGE

**asm/board.s -- Live hi-score update at Lscore_done:**

Insert a `csel`-based hi-score update at `Lscore_done` (line ~546), BEFORE the invisible mode check. Add immediately after the `Lscore_done:` label and its "END SCORING ENGINE" comment:

```asm

    adrp    x8, _hiscore@PAGE
    add     x8, x8, _hiscore@PAGEOFF
    ldr     w9, [x8]                   // current hiscore
    adrp    x10, _score@PAGE
    ldr     w11, [x10, _score@PAGEOFF] // current score

```

This uses the `hi` condition (unsigned higher) with `csel` to atomically update `_hiscore` whenever `_score` exceeds it. This runs on every lock (even non-clearing locks that add soft/hard drop points), so the display stays current. Registers w8-w11 are scratch at this point in the function (callee-saved registers are still live for the invisible mode check below).
