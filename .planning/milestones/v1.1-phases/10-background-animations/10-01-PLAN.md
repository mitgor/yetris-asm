**animation.s** -- Create new file `asm/animation.s` with these sections:

1. **File header**: Standard yetris-asm file header documenting the module.

2. **Constants** (in `__TEXT,__const` or as immediate values):
   - `ANIM_WIDTH` = 80, `ANIM_HEIGHT` = 24, `ANIM_CELLS` = 1920 (80*24)
   - Fire grayscale string: ` .':-=+*#%@#` (12 chars, indices 0-11)
   - Fire update rate: 100 (ms)

3. **Dispatch table** (in `__DATA,__const`):
```asm
.p2align 3
_anim_update_table:
    .quad _anim_fire_update_and_draw
    .quad _anim_water_update_and_draw
    .quad _anim_snakes_update_and_draw
    .quad _anim_life_update_and_draw
```

4. **`_anim_select_random`** -- Called once at startup:
   - Call `_arc4random_uniform(4)` to get w0 in 0-3
   - Store w0 to `_anim_type`
   - Call `_anim_init` for the selected animation
   - Return

5. **`_anim_init`** -- Initialize the selected animation's buffers:
   - Load `_anim_type`, dispatch to `_anim_fire_init` / stubs for others
   - Fire init: zero `_anim_buf1` (intensity buffer). Create cooling map in `_anim_buf2`: for each cell, call `_arc4random_uniform(14)` to get random 0-13, store as halfword. Then smooth 10 times: for each interior cell, average with 4 neighbors.
   - Water/Snakes/Life init: just return (stubs -- plan 10-02 fills these in)

6. **`_anim_dispatch`** -- Called from render.s and menu.s:
   - Receives x0 = WINDOW* (the parent window to draw into)
   - Save x0 to callee-saved register (x19)
   - Load `_anim_type`, index into `_anim_update_table`, call the function pointer passing x0=WINDOW*
   - Each animation function handles its own timer check internally

7. **`_anim_fire_update_and_draw(WINDOW* win)`** -- Complete fire algorithm:
   - **Timer check**: Call `_get_time_ms`, compare with `_anim_last_update`. If elapsed < 100ms, skip update but STILL draw. Otherwise, update timer.
   - **Update logic** (only when timer fires):
     - Pick `cooling_ratio` = `_arc4random_uniform(10) + 3` (range 3-12). 10% chance override to 1 (burst). 12% chance override to 30 (dim).
     - Bottom row (row 23): for each col 0-79, set `buf1[23*80+col]` = `_arc4random_uniform(11) + 90` (range 90-100). Store as signed halfword.
     - Sparks: for each col 0-79, call `_arc4random_uniform(433)`. If result < 10 (2.31% chance), inject `_arc4random_uniform(11)+90` at row `23 - _arc4random_uniform(4) - 3` (3-6 rows from bottom).
     - Propagate upward: for row 0 to 22 (row < height-1), for each col: `intensity = buf1[(row+1)*80+col] - cooling_ratio - buf2[row*80+col]`. If intensity < 0, clamp to 0. Store to `buf1[row*80+col]`. Use signed halfword load (`ldrsh`) and store (`strh`).
   - **Draw** (every call): For each cell (row 0-23, col 0-79):
     - Load intensity from `buf1[row*80+col]` (signed halfword)
     - If intensity <= 20, skip (draw nothing -- parent was erased so it's blank)
     - Map intensity to char index: `(intensity * 12) / 101` to get 0-11 index into grayscale string
     - Map intensity to color: >80 = COLOR_PAIR(3)|A_BOLD (white bold), >60 = COLOR_PAIR(1)|A_BOLD (yellow bold), >40 = COLOR_PAIR(1) (yellow), >20 = COLOR_PAIR(6)|A_BOLD (red bold), else = COLOR_PAIR(6) (red)
     - Call `_wattr_on(win, attr, NULL)`, `_mvwaddch(win, row, col, char)`, `_wattr_off(win, attr, NULL)`
   - **Register plan for fire update+draw:**
     - x19 = WINDOW* (callee-saved)
     - x20 = `_anim_buf1` base pointer
     - x21 = `_anim_buf2` base (cooling map)
     - w22 = row counter
     - w23 = col counter
     - w24 = width constant (80)
     - w25 = height constant (24)
     - x26 = grayscale string pointer

8. **Stubs for water/snakes/life** (just `ret` -- plan 10-02 implements):
```asm
_anim_water_update_and_draw:
_anim_snakes_update_and_draw:
_anim_life_update_and_draw:
    ret
```

**Important implementation notes:**
- All `mvwaddch` calls: the char argument is `w3` (4th arg, not variadic on ARM64 since `mvwaddch` takes fixed params: WINDOW*, int y, int x, chtype ch).
- COLOR_PAIR(n) = `n << 8`. So COLOR_PAIR(1) = 0x0100, COLOR_PAIR(3) = 0x0300, COLOR_PAIR(6) = 0x0600. A_BOLD = 0x00200000.
- Halfword buffer addressing: cell at (row, col) = base + (row * 80 + col) * 2 bytes.
- The cooling map smoothing in init can use a simple averaging loop: for each interior cell (1..78, 1..22), set cooling[r*80+c] = (cooling[(r-1)*80+c] + cooling[(r+1)*80+c] + cooling[r*80+c-1] + cooling[r*80+c+1]) / 4.
- Make all function labels `.globl` so they're linkable.
- The fire "draw every cell" loop can be optimized by batching attr_on/attr_off per row, but start simple (per-cell attr on/off) and it will work.
