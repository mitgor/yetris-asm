**Water animation (`_anim_water_update_and_draw`):**

Function signature: x0 = WINDOW*. Save to x19 (callee-saved).

1. **Timer check (300ms):** Call `_get_time_ms`. Load `_anim_last_update`. If elapsed < 300, skip to draw. Otherwise store new timestamp.

2. **Update (when timer fires):**
   - Swap buf1 and buf2 conceptually: use a flag byte or swap pointers. Simplest approach: maintain a `_anim_water_swap` flag byte (0 or 1). Toggle each update. When flag=0, read from buf1 write to buf2. When flag=1, read from buf2 write to buf1. Alternatively, just swap adrp-loaded pointers in registers -- simpler at the asm level: load both base addresses, do `swap x20, x21` (or just load in swapped order based on flag).
   - Random ripple injection: call `_arc4random_uniform(323)`. If result == 0 (0.31% chance per frame, not per cell), inject 90 at a random position: `_arc4random_uniform(80)` for col, `_arc4random_uniform(24)` for row. Store 90 as halfword to write-buffer at that position.
   - Wave propagation: For interior cells (row 1..22, col 1..78):
     ```
     new[row*80+col] = ((old[(row-1)*80+col] + old[(row+1)*80+col] + old[row*80+col-1] + old[row*80+col+1]) >> 1) - new[row*80+col]
     ```
     Clamp result to [0, 100]. Use signed halfword operations (`ldrsh`, `strh`). If result < 0, clamp to 0. If result > 100, clamp to 100.

3. **Draw:** For each cell (row 0-23, col 0-79):
   - Load height from the WRITE buffer (the one just updated)
   - Grayscale string (11 chars): `#@%#*+=-:'.` -- index = `(height * 11) / 101`
   - Color by height: >80 = COLOR_PAIR(3)|A_BOLD (white bold), >60 = COLOR_PAIR(2)|A_BOLD (cyan bold), >40 = COLOR_PAIR(2) (cyan), >20 = COLOR_PAIR(4) (blue), else = COLOR_PAIR(4)|A_BOLD (blue bold). Skip cells with height <= 0.
   - Call `_wattr_on`, `_mvwaddch`, `_wattr_off` per cell (same pattern as fire)

**Water init (`_anim_water_init`):**
- Fill buf1 with random 0-13: for each of 1920 cells, `_arc4random_uniform(14)`, store as halfword
- Fill buf2 with random 0-25: for each of 1920 cells, `_arc4random_uniform(26)`, store as halfword
- Set `_anim_water_swap` flag to 0 (add this byte variable near other anim state in animation.s's local data section, or add to data.s)

**Game of Life animation (`_anim_life_update_and_draw`):**

Function signature: x0 = WINDOW*. Save to x19.

1. **Timer check (200ms):** Same pattern. Elapsed < 200 -> skip update, still draw.

2. **Update (when timer fires):**
   - Read from buf1 (current generation), write to buf2 (next generation)
   - For interior cells (row 1..22, col 1..78): count 8 neighbors (load from buf1, count non-zero halfwords).
     - If cell alive (buf1[r*80+c] != 0): survive if neighbors == 2 or 3, else die (write 0 to buf2)
     - If cell dead (buf1[r*80+c] == 0): born if neighbors == 3, else stay dead
     - Write 1 (alive) or 0 (dead) as halfword to buf2
   - Edge cells (row 0, row 23, col 0, col 79): copy as-is or set to 0 (exclude from update per C++ pattern)
   - After update loop: swap buf1 and buf2 conceptually. Simplest: copy buf2 to buf1 (memcpy 3840 bytes -- simple loop or use `ldp`/`stp` blocks). OR use same swap-flag approach as water. Since only one animation runs at a time, can share the swap flag byte.

3. **Draw:** For each cell:
   - If alive (buf1[r*80+c] != 0 -- use the CURRENT buffer after swap): draw '#' with COLOR_PAIR(1) (yellow)
   - If dead: skip (background already erased)
   - Call `_wattr_on`, `_mvwaddch`, `_wattr_off` per living cell

**Life init (`_anim_life_init`):**
- 20% random fill: for each of 1920 cells in buf1, `_arc4random_uniform(5)`. If result == 0, store 1 (alive). Else store 0.
- Zero buf2 entirely

**Wire inits:** In the `_anim_init` dispatch (from plan 10-01), add:
- Type 1 -> `bl _anim_water_init`
- Type 3 -> `bl _anim_life_init`

**Implementation notes:**
- The water swap-flag approach: add a `.byte` in animation.s's data section: `_anim_buf_swap: .byte 0`. Toggle between 0 and 1 each update. When 0: read=buf1, write=buf2. When 1: read=buf2, write=buf1.
- GoL neighbor counting: 8 loads per cell (8 halfwords from buf1 at offsets: -81, -80, -79, -1, +1, +79, +80, +81 in halfword indices, so byte offsets: -162, -160, -158, -2, +2, +158, +160, +162). Count how many are non-zero.
- GoL swap: after writing to buf2, swap the flag so next time buf2 becomes read-buffer and buf1 becomes write-buffer. For draw, read from the just-written buffer.

**Snakes animation (`_anim_snakes_update_and_draw`):**

Function signature: x0 = WINDOW*. Save to x19.

The snakes animation has TWO independent timers:
- **Move timer** (`_anim_last_update`): every 50ms, move all snakes down
- **Add timer** (`_anim_last_add`): every 100-300ms random interval, add new snake(s)

1. **Move timer check (50ms):** Call `_get_time_ms`. Compare with `_anim_last_update`. If elapsed >= 50:
   - Store new timestamp to `_anim_last_update`
   - For each active snake i (0 to `_anim_snake_count - 1`):
     - Load snake struct: `_anim_snakes[i*4]` = {x: byte, y: signed byte, size: byte, pad: byte}
     - Increment y by 1 (snake falls down)
     - Check removal: if `(y - size) >= 24` (entire snake below screen), remove via swap-with-last:
       - Copy snake[count-1] to snake[i] (4-byte copy: `ldr w, str w`)
       - Decrement `_anim_snake_count`
       - Do NOT increment i (re-check this index since it now has a different snake)
     - Else: store updated y back, increment i

2. **Add timer check:** Load `_anim_last_add`. Compare with current time. If elapsed >= 200 (use 200ms as middle of 100-300 range, or pick random next interval):
   - Store current time to `_anim_last_add`
   - Check if `_anim_snake_count < 50`. If so:
     - Add new snake: x = `_arc4random_uniform(79) + 1` (1..79), y = negative value or 0..3 (`_arc4random_uniform(4)`), size = `_arc4random_uniform(13) + 2` (2..14)
     - Store to `_anim_snakes[count*4]`: byte x, signed byte y, byte size, byte 0 (pad)
     - Increment `_anim_snake_count`
   - 25% burst chance: `_arc4random_uniform(4)`. If result == 0, add 3-5 extra snakes (`_arc4random_uniform(3) + 3`), each with random x/y/size as above, up to max 50 total.

3. **Draw:** For each active snake i:
   - Load x, y (signed), size
   - Draw head: if y >= 0 and y < 24, call `_mvwaddch(win, y, x, '@')` with COLOR_PAIR(5)|A_BOLD (green bold)
   - Draw body: for j = 1 to size-1, body_y = y - j. If body_y >= 0 and body_y < 24, call `_mvwaddch(win, body_y, x, 'o')` with COLOR_PAIR(5) (green)
   - Use `_wattr_on`/`_wattr_off` to set color before/after each segment

**Snakes init (`_anim_snakes_init`):**
- Set `_anim_snake_count` to 1
- Create first snake: x = `_arc4random_uniform(79) + 1`, y = 0, size = `_arc4random_uniform(13) + 2`
- Store to `_anim_snakes[0]`
- Initialize `_anim_last_add` to current time (`_get_time_ms`)
- Initialize `_anim_last_update` to current time

**Wire init:** In `_anim_init`, type 2 -> `bl _anim_snakes_init`

**Register plan for snakes update+draw:**
- x19 = WINDOW*
- x20 = `_anim_snakes` base pointer
- w21 = loop index i
- w22 = `_anim_snake_count` current value
- w23 = snake x
- w24 = snake y (signed)
- w25 = snake size
- x26 = scratch for struct address computation

**Key detail: signed byte for y.** Snake y can become large positive (well past 24) as snakes fall off screen. Using unsigned byte is fine since y only grows from 0 upward; the removal condition `(y - size) >= 24` handles cleanup. Store y as unsigned byte, compute `(y - size)` in a register, compare. Max y before removal = 24 + max_size = 24 + 14 = 38, which fits in unsigned byte (0-255).

Actually, revisiting: y starts at 0-3 and only increments. size is 2-14. The snake is off-screen when `y - size >= 24`, so max y = 24 + 14 - 1 = 37. Unsigned byte is fine. Use `ldrb` for y loading and unsigned comparisons.
