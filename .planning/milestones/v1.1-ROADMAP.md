## Phase Details

### Phase 6: Subwindow Foundation
**Goal**: Game renders through named ncurses subwindows with pixel-perfect C++ panel layout
**Depends on**: Phase 5 (v1.0 complete)
**Requirements**: LAYOUT-01, LAYOUT-02, LAYOUT-03, LAYOUT-04, LAYOUT-05, LAYOUT-06, LAYOUT-07
**Success Criteria** (what must be TRUE):
  1. Game screen displays 5 distinct panels (hold, score, board, next, statistics) in separate subwindows with visible borders at the exact column positions of the C++ original
  2. Menu screen renders logo and menu items through dedicated subwindows matching C++ menu layout
  3. Hold panel shows current held piece with "Hold" title, score panel shows Score/Level/Lines labels, next panel shows next piece with "Next" title, statistics panel shows piece counts and timer with "Statistics" title
  4. Game is fully playable with no visual artifacts -- all existing v1.0 functionality works identically through the new subwindow rendering
**Plans**: 2 plans

Plans:
- [ ] 06-01-PLAN.md — Window infrastructure: WINDOW* data slots, layout.s lifecycle functions, Makefile update
- [ ] 06-02-PLAN.md — Convert all rendering to subwindows, wire lifecycle into state machine

### Phase 8: Modern Scoring Engine
**Goal**: Scoring matches modern Tetris guideline with combos, T-spins, and all bonus systems
**Depends on**: Phase 6
**Requirements**: SCORE-01, SCORE-02, SCORE-03, SCORE-04, SCORE-05, SCORE-06, SCORE-07, SCORE-08
**Success Criteria** (what must be TRUE):
  1. Line clears award level-multiplied points (Single=100*lvl, Double=300*lvl, Triple=500*lvl, Tetris=800*lvl) and the score panel updates immediately
  2. Consecutive line-clearing placements accumulate a combo counter that adds 50*combo*level bonus points; the counter resets when a piece locks without clearing lines
  3. T-piece placements after rotation with 3+ diagonal corners occupied are detected as T-spins and award T-spin scoring values (400-1600 * level); consecutive difficult clears (Tetris or T-spin) receive a 1.5x back-to-back bonus
  4. Clearing all blocks from the board triggers a perfect clear bonus; soft drop awards 1 point per cell, hard drop awards 2 points per cell
**Plans**: 2 plans

Plans:
- [ ] 08-01-PLAN.md — Line-clear scoring engine: level multiply, combo, back-to-back, perfect clear
- [ ] 08-02-PLAN.md — T-spin detection and scoring, soft drop and hard drop scoring

### Phase 9: Line Clear Animation

**Depends on**: Phase 6, Phase 8

### Phase 11: Hi-Score Persistence
**Goal**: Top score survives across game sessions via file storage
**Depends on**: Phase 8
**Requirements**: HISCORE-01, HISCORE-02, HISCORE-03
**Success Criteria** (what must be TRUE):
  1. When the player achieves a new high score and the game ends, the score is saved to ~/.yetris-hiscore and persists across application restarts
  2. On startup, the previous high score loads from disk and displays in the score panel with "Hi-Score" label; if no file exists or is unreadable, it displays "(none)" with a value of 0
**Plans**: 1 plan

Plans:
- [ ] 11-01-PLAN.md — Hiscore load/save via Darwin syscalls, conditional score panel display, live tracking
