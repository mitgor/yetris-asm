# Requirements: yetris-asm

**Defined:** 2026-02-26
**Core Value:** A fully playable Tetris clone in ARM64 assembly that is measurably smaller and faster than the C++ original, with documented research findings.

## v1 Requirements

Requirements for initial release. Each maps to roadmap phases.

### Foundation

- [x] **FOUN-01**: AArch64 assembly project builds and runs on macOS Apple Silicon using Apple `as` and `ld`
- [x] **FOUN-02**: Binary links against system ncurses and initializes/cleans up terminal correctly
- [x] **FOUN-03**: Darwin ABI conventions are correct (x16 syscalls, svc #0x80, x18 reserved, 16-byte stack alignment, underscore prefixes)
- [x] **FOUN-04**: Makefile builds assembly source files in asm/ directory alongside existing C++ in src/

### Core Mechanics

- [x] **MECH-01**: 7-tetromino set (I, J, L, O, S, T, Z) defined as data tables with 4 rotations each
- [x] **MECH-02**: SRS rotation system with full wall kick tables (5-kick test per rotation, separate I-piece table)
- [x] **MECH-03**: 10x20 playfield with collision detection
- [x] **MECH-04**: Gravity timer scaling from level 1 (1000ms) to level 22 (0ms)
- [x] **MECH-05**: Piece locking when piece cannot move down
- [x] **MECH-06**: Line clearing with gravity drop on cleared rows
- [x] **MECH-07**: Hard drop (instant) and soft drop (accelerated)
- [x] **MECH-08**: 7-bag random piece generator
- [x] **MECH-09**: Next piece preview (1-7 configurable)
- [x] **MECH-10**: Hold piece mechanic (one slot, can't re-hold until lock)
- [x] **MECH-11**: Ghost piece (landing preview)
- [x] **MECH-12**: Game over detection (piece spawns in occupied space)
- [x] **MECH-13**: Pause and resume with timer suspension
- [x] **MECH-14**: Level progression based on lines cleared
- [x] **MECH-15**: Scoring (1-line=100, 2=300, 3=500, 4=800, +10 per piece lock)

### Rendering

- [x] **REND-01**: ncurses-based terminal rendering of board, current piece, and UI panels
- [x] **REND-02**: Color-coded pieces (7 distinct colors matching yetris: S=green, Z=red, O=yellow, I=cyan, L=orange, J=blue, T=magenta)
- [x] **REND-03**: Score, lines, and level display panel
- [x] **REND-04**: Piece statistics panel (count per type + singles/doubles/triples/tetris)
- [x] **REND-05**: Input handling (arrow keys, space for hard drop, configurable keys)

### UI

- [x] **UI-01**: Main menu with game options
- [x] **UI-02**: Help screen with keybinding reference
- [x] **UI-03**: Starting level selection (1-22)
- [x] **UI-04**: Game mode options (initial noise, invisible mode, ghost/hold toggles)

### Optimization Research

- [x] **OPT-01**: Register-only state packing — compact game flags (pause, game_over, can_hold, level, score) in callee-saved registers x19-x28
- [x] **OPT-02**: NEON/SIMD board operations — vectorized collision detection using 128-bit v-registers on bitfield board layout
- [x] **OPT-03**: NEON/SIMD line detection — vectorized full-row scan (AND all 10 bits, compare to 0x3FF)
- [x] **OPT-04**: Syscall batching — batch frame output to minimize kernel transitions
- [x] **OPT-05**: Binary size optimization — dead_strip, symbol stripping, section analysis, comparison to C++ baseline
- [x] **OPT-06**: Each optimization technique measured before/after with quantitative results

### Measurement and Documentation

- [x] **MEAS-01**: Binary size tracked at each development stage vs C++ yetris baseline
- [x] **MEAS-02**: Frame timing measurements using mach_absolute_time or clock_gettime
- [x] **MEAS-03**: CPU profiling with Instruments to identify hotspots
- [x] **MEAS-04**: Research writeup documenting all techniques, measurements, tradeoffs, and findings
- [x] **MEAS-05**: ARM64-specific vs generic assembly optimizations distinguished in writeup

## v2 Requirements

Deferred to future release. Tracked but not in current roadmap.

### Visual Enhancements

- **VIS-01**: Background animations (fire, water, snakes, game-of-life) ported from C++
- **VIS-02**: Configurable animation selection for menu and in-game

### Persistence

- **PERS-01**: High score persistence to file
- **PERS-02**: Profile system with per-user settings and keybindings
- **PERS-03**: Configurable keybinding persistence

### Advanced Research

- **ADVR-01**: Raw PMU cycle counter access via kperf API
- **ADVR-02**: NEON line-clear gravity compaction (row removal via SIMD shuffle)
- **ADVR-03**: Energy profiling via powermetrics

## Out of Scope

| Feature | Reason |
|---------|--------|
| x86/Intel assembly | ARM64 only — targeting Apple Silicon |
| GUI / graphical rendering | Terminal-only via ncurses |
| Cross-platform (Linux/Windows) | macOS Darwin only (ABI, Mach-O, syscall conventions) |
| Rewriting ncurses | Use system library as-is |
| T-spin detection/scoring | yetris doesn't implement it; not in scope for faithful clone |
| Back-to-back bonus scoring | yetris uses simplified scoring model |
| Multiplayer / garbage lines | Single-player only |
| Static linking ncurses | Must dynamically link system ncurses on macOS |
